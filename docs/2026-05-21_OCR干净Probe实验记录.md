# 2026-05-21 OCR干净Probe实验记录

## 本轮先解决的不是识别率，而是实验污染

这轮先修掉了 3 类会污染 OCR 结论的链路问题：

1. `focus_chain` 误把视觉焦点内容写成文本/内部项，污染记忆召回。
2. probe 第二个 tick 开始会把上一 tick 的 `prediction` 文本重新混入召回查询。
3. probe 观测时仍继续写 `exact_external` / `focus_chain`，导致测试样本“边测边学”。

修复后：

- probe 查询改为“外源优先”，并暴露 `recall_query_preview`
- probe 默认关闭 `runtime_write_focus_chain_memory`
- probe 默认关闭 `runtime_write_exact_external_memory`
- `Bn` 顶部记忆已回到 `exact_external`，不再被 `focus_chain` 垄断

## 当前阶段的严谨结论

可以确认：

- 之前较好的 OCR 结果，部分确实受到了实验污染影响
- 当前 V2 视觉链条已经能跑通“纯视觉 probe -> 记忆召回 -> C* 预测”
- 但在干净 probe 条件下，`3/8` 的 AP-native 视觉识别仍未被稳健证明

当前真正剩下的主问题已经收敛到两点：

1. 视觉特征的跨变体可分辨性还不够稳
2. `C*` 分支聚合对相似记忆仍偏敏感，容易被近邻共享结构带偏

## 下一步最该继续做的事

1. 继续增强视觉结构特征
   - 闭环/双环
   - 上下开口差异
   - 左右/上下质量分布
   - 曲率与对称的组合特征

2. 改 `C*` 聚合
   - 对近重复邻居去重
   - 对共享视觉特征降权
   - 对外源 `exact_external` 的文本项提高聚合可信度

3. 在“只读 probe”基线上继续跑更大的对照实验，避免再次被假阳性误导
