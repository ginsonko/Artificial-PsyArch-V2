# AP V2 对 Claude 审计与常见误解的纠偏说明

日期：2026-05-25  
适用对象：第一次接触 AP / PA 项目、刚读完一期论文、或刚看过 `项目审计报告2.md` 的外部审阅者  
定位：**这不是辩护稿，也不是营销稿，而是一份尽量严谨、可复核、分清已证据 / 未证据 / 误解边界的纠偏说明**

---

## 1. 文档目的

这份文档要解决的，不是“别人批评 AP 我们要不要反击”，而是一个更基础的问题：

> 当审阅者把一期论文、一期最小原型、二期原型、默认配置、局部规则层、以及长上下文 AI 的外推性批判混在一起时，很容易在错误前提上得出过强结论。

因此本文目标有四个：

1. 明确指出外部审计材料《项目审计报告2.md》中哪些批评成立，哪些推论过度，哪些属于读错对象、错把局部当整体。
2. 用当前二期仓中的真实代码、实验报告与论文现稿，解释 AP 的能力主张到底来自哪里。
3. 说明为什么“有先天规则”不等于“整个系统只是纯规则 if-then 专家系统”。
4. 给出一套更公平、更可复现的审阅顺序和实验规范，避免用无效实验直接否定系统。

---

## 2. 先给结论：哪些批评成立，哪些不成立

### 2.1 这些批评是成立的，而且应该认真接受

1. **工程实现仍然不够优雅**。  
   当前二期仓是快速演化中的实验原型，局部硬编码、参数堆叠、早期设计痕迹、命名与路径不够整洁，都是真问题。

2. **不是成品，也还远远不是“已经成熟的通用智能系统”**。  
   当前最合理定位，是“可复现实验原型 + 白箱认知闭环工程底盘”，不是完整产品，不是已经终局定型。

3. **有些论文或对外表述需要更谨慎**。  
   尤其是当一期论文主要基于最小原型，而二期已经前进较多时，旧表述容易被误读成“已经全部工程实现完毕”。这一点确实需要修订。

4. **当前能力证据大多还是“雏形证据”“机制证据”“原理可行性证据”**。  
   例如多模态统一绑定、动态视觉对象层、认知感受闭环、音频-文本反向联想，都有初步或较强证据，但还不能夸大成“开放世界稳定成熟”。

5. **不能把当前 AP V2 说成“已经可以替代所有 LLM 用途”**。  
   当前二期本来就不是这个定位。论文现稿里也没有这个说法。

### 2.2 这些判断则是不成立或明显过度的

1. **“AP 本质上只是那份 YAML 规则文件，所以没有任何后天学习能力。”**
2. **“代码里 if 多，所以整个认知算法本质就是纯规则专家系统。”**
3. **“没有神经网络权重矩阵，所以不可能形成任何逻辑/直觉雏形。”**
4. **“一期最小原型 / 默认旁路路径 / 某个局部模块的实现方式，可以直接定义整个 AP 架构。”**
5. **“论文或仓库宣称了‘完全替代 LLM’、‘自带神经网络’，所以作者在做虚假宣传。”**

这些结论的问题，不是“观点太负面”，而是：

- 它们混淆了**架构层**与**实现层**；
- 混淆了**先天规则层**与**主认知闭环**；
- 混淆了**一期最小原型**与**当前二期仓**；
- 混淆了**局部工程问题**与**架构能力是否存在**；
- 还混入了若干并未出现在论文或当前二期材料中的外推说法。

## 3. 审计报告的核心误区是什么

`项目审计报告2.md` 全文成立的关键前提，其实是一个先入为主的判断：

> **它默认认为“先天规则编码”就是 AP 产生学习、认知感受和心智能力的根本来源。**

在这个前提下，它后续所有推理就会自然变成：

- 如果感受器后面有规则，那感受就是规则写死的；
- 如果奖励和惩罚要靠先天规则触发，那学习就是规则在演戏；
- 如果清空一部分规则后某些能力消失，那能力的来源一定全在规则里；
- 如果没有神经网络权重矩阵，那就只能是 1990 年代符号 AI 或 N-gram。

问题在于，这个前提本身就不完整。

### 3.1 先天规则层在 AP 里是什么

先天规则层在 AP 中的作用，更接近：

1. 先天驱动力与先天反应阈值
2. 某些底层“什么时候值得注意、什么时候应奖惩、什么时候应保护自己”的白箱门控
3. 一些为了冷启动和拟人持续运行而保留的底层调制逻辑

也就是说，它更像：

- 生物中的先天偏置
- 先天反射
- 某些固定的低层调节规则

而**不是**整套学习与抽象能力本身。

### 3.2 AP 真正的学习与认知能力主线，不在“规则层先触发了什么”

AP 二期当前的主线，是一条持续运行的认知闭环，而不是“规则判定 -> 直接输出答案”的单步专家系统。

当前 README 对主线的概括是：

`文本输入 -> SA 竞争 -> 状态池 -> R_state -> Bn -> C_i -> C* -> rules -> 执行器审计 -> A_focus -> 短期记忆 -> 长期记忆`

这里最关键的点是：

1. **规则层不在最前面，而是在中后段。**
2. 系统先经历：
   - 刺激进入
   - SA 竞争
   - 状态池组织
   - 显式记忆召回
   - 预测分支构造
   - 综合预测包形成
   - 预测与现实的错配追踪
3. 规则层更多是对这整套运行态进行**调制、审计、注入感受或驱动力**，而不是直接替代整条主线。

换句话说：

> **AP 的先天规则层很重要，但它不是“学习和抽象能力的本体”；学习与抽象能力的本体，来自完整认知闭环内部的查存一体、预测分支、状态池竞争、错配残差、跨 tick 连续性与记忆回写。**

## 4. 当前二期仓里，主认知闭环到底有什么

为了避免空泛，这里只列当前仓中可以直接检查的东西。

### 4.1 混合记忆召回，不是单一字符串匹配

[memory/memory_store_v2.py](../memory/memory_store_v2.py) 当前已经不是“纯 overlap demo”。

README 明确写到：

- `posting + bigram + 向量嵌入 + FAISS ANN + 时空邻域`

实际代码中，召回与预测分支构造会综合多种因素，例如：

- `unit_overlap`
- `bigram_overlap`
- `successor_bias`
- `reality_bonus`
- `spacetime_score`
- `vector_related`

尤其在 `build_prediction_branches(...)` 中，当前分支邻接分数是多个因素的组合，而不是 `if apple then banana` 这种硬写关系。

这意味着什么？

这意味着当前二期里，至少有一部分“关系建立”不是由手工写死的目标对象映射来完成，而是由：

1. 当前查询结构
2. 历史记忆结构
3. 时空邻接关系
4. 后继优势
5. 向量相关性

共同决定。

### 4.2 状态池里有真实的“预测-现实错配分解”

[core/state_pool_v2.py](../core/state_pool_v2.py) 中的 `_record_prediction_trace(...)` 并不是一个装饰性命名。

当前它会显式记录：

- `matched_labels`
- `unexpected_labels`
- `missed_predicted_labels`
- `predicted_mass`
- `actual_mass`
- `overprediction_mass`
- `underprediction_mass`
- `mismatch_mass`
- `committed_match_mass`
- `halo_labels`

这不是“情绪写死了所以叫惊讶/违和感”的那种命名包装，而是先有预测和现实的分解，再从这些可观察量导出感受与调制。

### 4.3 认知感受与奖励/惩罚并不是只靠单一阈值硬触发

当前二期中，至少有一条明确可查的链路是：

1. 预测与现实的错配 / 对齐先在状态池层形成可量化量
2. 这些量进入 IESM 指标与 runtime 认知平衡
3. 再进一步转为：
   - `surprise`
   - `dissonance`
   - `correctness`
   - `grasp`
   - `expectation`
   - `pressure`
4. 再进一步转为内在奖惩信号

[iesm/rules_engine_v2.py](../iesm/rules_engine_v2.py) 里已有：

- `state.prediction_alignment_score`
- `state.prediction_overprediction_ratio`
- `state.prediction_underprediction_ratio`
- `state.prediction_grasp_score`
- `state.prediction_mismatch_balance`

[core/runtime_v2.py](../core/runtime_v2.py) 又进一步把这些量接到：

- `correctness_delta_reward`
- `grasp_delta_reward`
- `surprise_recovery_reward`
- `dissonance_recovery_reward`
- `dissonance_delta_punishment`
- `surprise_delta_punishment`
- `expectation_tonic_reward`
- `pressure_tonic_punishment`

所以更准确的说法应该是：

> 当前 AP V2 的认知感受链，不是“没有预测闭环，纯靠规则写情绪”，而是“预测闭环和状态池错配先存在，规则层再读取这些量做调制与白箱注入”。

这和“纯规则系统”的差别非常大。

## 5. 为什么“有规则”不等于“纯规则系统”

这是争论里最容易滑坡的一点。

### 5.1 所有复杂系统几乎都有规则层

只要系统要长期运行、要做安全约束、要做输入筛选、要做阈值保护、要做冷启动边界，就一定会有规则层。

Transformer 工程实现里也有：

- 分支逻辑
- 截断逻辑
- 工具调度规则
- 记忆裁剪策略
- 安全审查策略

不能因为代码里有 `if`，就把整个系统定性为“if-then 专家系统”。

### 5.2 真正该问的是：规则住在哪里，主认知更新住在哪里

当前二期里，更合理的分层是：

1. **感受器层**：把文本、图像、音频变成可入池的刺激结构
2. **状态池层**：组织当前认知场
3. **记忆层**：进行显式召回、时空近邻召回、向量相似召回
4. **预测层**：构造 `Bn / C_i / C*`
5. **错配层**：记录预测与现实差异
6. **感受/奖惩/规则层**：根据运行态做调制和反馈
7. **记忆回写层**：把本轮结果沉入长期或短期结构

如果一个系统的核心运行都发生在 1、2、3、4、5，而规则主要在 6，那么它最多应被称为：

> **强先天门控的白箱混合认知闭环**

而不是“纯规则系统”。

### 5.3 审计报告中的“清空 YAML”实验，为什么不能直接推出“所有学习能力都不存在”

`项目审计报告2.md` 最强的一条，是它做了“清空 YAML 后很多认知/情绪/行动指标归零”的实验。

这条实验不是毫无价值，它说明了两件事：

1. 当前那份被审计对象里，先天规则层确实承载了大量显式的感受/行动调制职责。
2. 一期/旧路径/默认配置中的某些链路，对规则层依赖很重。

但它**不能自动推出**：

1. 当前二期主认知闭环没有学习能力；
2. 结构记忆、预测分支、状态池竞争这些也都只是装饰；
3. AP 的抽象、召回、顺序偏置、跨模态绑定全部都是假的。

因为“清空规则”做掉的是大量门控和调制层，不等于把完整认知闭环的所有有效路径都单独隔离验证过了。

更公正的做法应该是分别问：

1. 清空规则后，**结构记忆与预测分支**还剩什么？
2. 清空规则后，**文本顺序偏置与后继优势**还剩什么？
3. 清空规则后，**多模态绑定的记忆痕迹**还剩什么？
4. 清空规则后，**认知感受与奖惩调制**还剩什么？

这些问题要分开，不该一把梭哈成“全部能力=YAML”。

## 6. AP 当前为什么敢宣称自己有这些能力雏形

注意，这里说的是“为什么敢宣称有雏形”，不是“为什么已经完全证明成熟”。

### 6.1 文本连续性、同义桥接、局部抽象

[docs/V2_阶段性实验评估报告_20260520.md](./V2_阶段性实验评估报告_20260520.md) 当前给出的阶段性证据是：

1. 纯文本环境下出现了“连续可读、局部有逻辑”的短跨度语言组织雏形
2. 在预测-实际错配场景里出现了同义近义桥接
3. 基于共同部分的实时抽象信号已经出现

这并不等于“已经是成熟文本生成模型”，但它足以反驳“没有神经网络所以不可能出现任何逻辑/顺序/局部生成雏形”的绝对化说法。

### 6.2 原生视觉 OCR-like 雏形

[docs/V2_原生视觉文字识别_OCR_like_实验报告_2026-05-22.md](./V2_原生视觉文字识别_OCR_like_实验报告_2026-05-22.md) 当前支持的是：

1. 图像和文本共现训练后，图像能单独拉起对应文本
2. 在旧上下文干扰下，系统可以跨数个 tick 完成从旧对象到新对象的认知翻转

这说明它不是只会一次性静态匹配，而是已经带有跨 tick 的认知更新特征。

### 6.3 原生音频 OCR-like 雏形

[docs/V2_原生音频识别_OCR_like_实验报告_2026-05-23.md](./V2_原生音频识别_OCR_like_实验报告_2026-05-23.md) 当前支持的是：

1. 音频结构与文本可被绑定
2. 训练后，audio-only 能逐步把目标文本拉起
3. text-only 能反向带起与文本关联的音频结构标签

这说明 AP 的多模态统一并不只停留在“图片和文字”，而已经开始向“音频-文字”的统一记忆生长。

### 6.4 多模态统一对象教学与反向联想

[docs/V2_多模态教学与海豚训练综合实验报告_2026-05-23.md](./V2_多模态教学与海豚训练综合实验报告_2026-05-23.md) 当前给出的最重要证据是：

1. 图像、音频、文本可以在同一状态池中共同教成一个对象
2. 在保留稳定上下文时，单一模态线索可以触发其它模态的联想
3. 新旧对象切换时存在分层翻转，而不是单步硬切
4. 前端已经能把这种“内心画面 / 内心声音代理结构”展示出来

### 6.5 动态视觉对象层

[evidence/bundled/dynamic_visual_ocr_report_20260523/report.md](../evidence/bundled/dynamic_visual_ocr_report_20260523/report.md) 当前表明：

1. 运动本身已经可以作为对象级动态摘要进入系统
2. 动态视觉已经接入视线与注意力接口
3. 复杂场景下还没稳定完成最终 OCR 闭环，但动态处理链本身已经成立

这说明“通过相邻 N 个图区分对象运动与背景运动”的目标，不再只是口头设想，而已进入可实验的工程阶段。

### 6.6 认知感受与奖惩闭环

[docs/V2_认知感受与情绪变化实验报告_2026-05-22.md](./V2_认知感受与情绪变化实验报告_2026-05-22.md) 以及 [evidence/bundled/multichannel_feelings_20260523/report.md](../evidence/bundled/multichannel_feelings_20260523/report.md) 当前支持：

1. 惊、违和感、正确感、期待、压力等通道已有可观测运行态
2. 这些通道能进一步影响召回排序、奖惩和后续处理
3. 时间感、节奏感、反馈价性等也开始作为状态池中的一等公民进入主链

这意味着“认知感受是系统对自己认知过程本身的感受”这条哲学，不再只是名词，而已有白箱运行态支撑。

## 7. AP 的独特价值到底在哪里

AP 当前的独特价值，并不在于“立刻替代 LLM 做所有事情”。

更准确地说，它的价值在于尝试把下面这些通常被拆散、外挂化、黑箱化的部分，放回同一个持续运行闭环里：

1. 状态池
2. 统一刺激级查存一体
3. 预测与现实的错配跟踪
4. 认知感受
5. 情绪调制
6. 奖惩与注意力
7. 多模态统一绑定
8. 连续运行与回放审计

如果只从“是不是神经网络”这个角度看，它当然不是 Transformer。

但如果从“有没有尝试把长期运行主体最关键的那些底层机制重新组织成一套白箱闭环”这个角度看，它确实有明显的独特价值。

这个价值主要体现在三点：

### 7.1 白箱性

当前二期非常重视：

- run / tick 级回放
- live / sidecar / runtime 导出
- 规则、调参、情绪、动作、召回、预测分支的结构化观测

这让它和典型黑箱模型很不一样。

### 7.2 跨模态统一路径

图像、音频、文本不是分别丢给三个各自独立的任务模型，然后最后拼一下结果，而是尽量进入同一条 tick 主链。

### 7.3 持续运行主体视角

AP 不是先假定“我要生成一句完美回答”，而是先假定“系统需要长期运行、积累经验、感受错配、维持主线、调节注意、接受奖励惩罚，再在这个过程中逐步长出能力”。

这套视角与主流大模型路线是不同的，因此更适合被理解为：

> **一种认知底座 / 持续运行心智底盘 / 可和 LLM 协同的白箱长期认知核心**

而不是“另一个同质化文本大模型”。

## 8. 如何正确开始理解 AP，而不是一上来就看偏

推荐阅读顺序如下：

1. **先看论文现稿里的问题意识与完整闭环定义**
   - [paper/AP_manuscript_20260513.pdf](../paper/AP_manuscript_20260513.pdf)
   - 尤其看第 1 章与第 2 章

2. **再看当前二期 README 的工程主链**
   - [README.md](../README.md)

3. **然后看阶段性实验总评**

## 9. 如何公平复现实验

### 9.1 不要混淆“一期最小原型”“二期原型”“论文现稿”“旧审计报告”

这是最容易造成结论混乱的地方。

请至少分清：

1. 一期最小原型：用于证明最小主线与哲学雏形
2. 当前二期仓：已经有更完整的多模态、状态池、记忆、观测、反馈链
3. 论文现稿：以一期为主，但已经包含完整哲学和部分实验叙述
4. 审计报告：是某个版本、某个仓、某个预设前提下的分析，不等于整个 AP 项目

### 9.2 不要用低质量输入直接否定系统

这一点非常重要。

AP 当前是以“tick”为单位持续运行的认知原型。

如果一个审阅者在极少数 tick 内：

- 塞进远高于人类实际环境密度的大量文本
- 强求 1 到 2 tick 内必须完全稳定
- 完全不允许冷启动
- 把空 tick 稳定段全部砍掉
- 连续切换多个高干扰对象还要求零残留

那得到的不是“能力评估”，而更接近“过载压力测试”。

过载测试当然有价值，但它不能被直接解释成“架构不可能产生该能力”。

### 9.3 应区分四类实验

1. **机制存在性实验**：验证某条链路是否已经打通  
   例：图像能否单独拉起文本；文本能否反向拉起音频结构。

2. **边界实验**：验证当前何时失效  
   例：reset transient state 后召回是否变弱。

3. **压力实验**：验证复杂场景下是否仍稳定  
   例：动态视觉 clutter 场景。

4. **长期成熟度实验**：验证大规模、长时程、开放环境稳定性  
   这类当前多数还不能说已经完成。

如果你拿第 3 类或第 4 类的未通过，去否定第 1 类“机制存在性”，逻辑上就是不成立的。

## 10. 对 `项目审计报告2.md` 的逐点纠偏

这里只抓最关键的几类。

### 10.1 “论文/项目宣称完全替代 LLM、或自带神经网络”

这是报告中最明显的问题之一。

当前能直接核到的是：

1. 当前二期仓 README 没有“完全替代 LLM”的说法。
2. 用当前论文现稿检索：
   - 能找到“**并不替代所有感受器、执行器或语言模型**”
   - 能找到“**AP 作为长期认知核心，大语言模型作为解释器、翻译器、工具组织者和安全审查者**”
   - 找不到“完全替代当前 LLM”
   - 找不到“自带神经网络”
   - 找不到“比肩 LLM”

所以这类说法不是“作者真的这样写了”，而更像是审计者把某些外部聊天表述、旧说法、或者自己的外推混进了结论。

### 10.2 “清空 YAML 后很多指标归零，所以 AP 的心智本质上只有 YAML”

这条只成立一半。

成立的一半是：

- 说明规则层在被审计对象里承担了大量显式调制职责

不成立的一半是：

- 它不能自动推出完整主认知闭环没有独立价值
- 也不能推出所有学习、顺序偏置、跨模态绑定、状态池竞争都是假的

### 10.3 “代码里 if 很多，所以几乎所有算法都是 if-then”

这是一种常见的观察事实，但被扩展成了错误结论。

更准确的说法应该是：

1. AP 当前确实有很多规则、阈值、参数、分支逻辑
2. 但它的主认知链并不等于这些规则
3. 混合召回、状态池预测错配、跨模态绑定、连续翻转过程，都不是一句“if 多”就能概括掉的

### 10.4 “没有神经网络，所以不可能产生文本组织或直觉雏形”

这条结论也过头了。

“没有神经网络权重矩阵”只能推出：

- 它不是 Transformer
- 它不是典型可微分深度学习模型

不能推出：

- 不可能产生局部文本连续性
- 不可能产生顺序偏置
- 不可能产生基于预测错配的联想或桥接

因为这些现象是否存在，最终要看实验，不是看架构类别名称。

### 10.5 “HDB 有上限，所以架构整体上限已经被证明”

这也是范围外推。

即便某个记忆层实现、数据库规模或向量策略有工程上限，也只能说明：

- 当前实现有容量、效率、检索质量边界

不能直接推出：

- AP 这类架构的理论上限已经被证明

这两者不是一回事。

## 11. 论文与对外表述，确实还可以怎么改

为了公平，也必须说清楚应该改什么。

### 11.1 明确区分一期论文与二期工程现状

建议在显眼位置标注：

1. 论文主要基于一期最小原型与对应实验
2. 当前二期仓是更进一步的实验原型
3. 两者不能互相偷换

### 11.2 明确哪些是“已证明”，哪些是“雏形”，哪些是“目标”

例如：

- 已证明：某条链路存在
- 初步支持：某类复杂能力开始出现
- 尚未证明：开放环境长期稳定成熟度

### 11.3 避免让“哲学目标”读起来像“已经全部工程完工”

哲学表达本身没问题，问题是要配合：

- 当前实现状态
- 当前实验边界
- 下一阶段目标

### 11.4 更主动回应“规则层 vs 闭环主线”的误解

这是外界最容易误解的地方，完全可以在论文或仓库里单开一节解释：

1. 先天规则层是什么
2. 它不是什么
3. 主认知闭环在哪里
4. 哪些能力主张来自闭环，哪些来自规则调制

## 13. 最终结论

最公正的结论，不应该是“AP 已经赢了”，也不应该是“AP 只是 YAML 规则系统所以没有任何独特价值”。

更准确的说法是：

> **AP 当前是一个强先天门控、白箱化、持续运行的认知闭环实验原型。它当然还不成熟，工程上也不够优雅，很多高层能力还只是雏形；但把它简化成“纯规则 if-then 专家系统”“没有任何学习、抽象、联想能力”也是不成立的。**

它最值得严肃对待的，不是它已经做完了什么，而是它正在尝试用一条不同于主流黑箱模型的路径，把：

- 记忆
- 预测
- 错配
- 感受
- 奖惩
- 注意力
- 多模态
- 连续运行

这些本来常被拆散的部分，重新放回同一个可观测、可回放、可复核的认知主体闭环里。

这件事本身，就已经值得被公正理解，而不是被错误前提带偏。
