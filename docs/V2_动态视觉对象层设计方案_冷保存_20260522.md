这份文档用于冷保存我们已经达成一致的“动态视觉对象层”方案，避免后续因为上下文压缩、阶段切换、调参或实验压力而偏离预期。

## 2. 动态视觉要解决的真正问题

动态视觉的目标，不只是给每个视觉采样点多加一个 `motion` 数值。

它要解决的，是以下两件事：

1. 连续多个 tick 的视觉输入中，哪些局部结构属于“同一个持续存在的对象”。
2. 这些对象相对于周围场景的运动趋势，是否足以让系统更容易把它们认知为一个整体。

也就是说，动态视觉的核心不是“哪里变了”，而是：

- 哪些部分以相近的方式持续变化
- 哪些边界两侧存在明显的相对速率差
- 哪些对象在多个 tick 上可以被连续、稳定地追踪

---

## 3. 与整体哲学的一致性

### 3.1 不把动态视觉做成外挂视频算法

动态视觉仍然属于 AP 的感受器内部加工层，而不是独立外挂识别器。

它的地位应与文本残响相似：

- 近期输入先在感受器内部形成短时缓存
- 该缓存不直接等价于状态池中的显式对象
- 经由内部比较与整理后，只把少量高价值摘要送入统一认知链

### 3.2 不做对象类别硬编码

动态视觉对象层不应该写死“这是人”“这是车”“这是球”。

它应只使用连续量与白箱规则：

- 形状相似
- 位置连续
- 尺寸变化平滑
- 方向与速度连续
- 边界内外残差速度差
- 相对视焦点位置关系

### 3.3 不破坏统一状态池哲学

动态视觉可以在感受器内部有自己的短时缓存与轨迹维护，但这不意味着另起一个认知系统。

正式口径应是：

- 感受器内部允许存在短时加工层
- 统一状态池只接收提炼后的可认知对象
- 不把所有视觉历史原样倒入状态池

## 4. 当前 `VisionSensorV1` 的真实基础

当前视觉链路已经具备以下基础：

1. `raw_samples`
   - 原图分辨率稀疏采样
   - 已带局部亮度、边缘、直曲、开口、对称、位置、局部 motion 等连续通道

2. `memory_write_samples`
   - 面向焦点记忆的少量高价值视觉特征

3. `global_structure_samples`
   - 面向整图前景的全局结构摘要
   - 当前更像“整图一个整体前景摘要”，还不是多对象切分

4. `_prev_raw_samples`
   - 已有前一帧级别的局部 motion 比较缓存

5. `_fixation_buffer`
   - 已有注视累积型短时缓存

因此，动态视觉不是从零开始做，而是在现有静态视觉和局部 motion 之上，再补一层“对象级的时间连续性建模”。

---

## 5. 正式架构分层

推荐把视觉感受器理解为四层：

### 5.1 L0：原始稀疏采样层

职责：

- 从原图分辨率中进行偏置采样
- 输出 `raw_samples`
- 强调边缘、高对比度、视焦点附近、运动变化区

这层负责“获得足够多的局部证据”。

### 5.2 L1：静态结构对象候选层

职责：

- 从当前 tick 的视觉证据中，生成有限个形状对象候选
- 候选对象不是语义标签，而是连续结构描述

输出：

- `shape_candidates`

### 5.3 L2：短时动态轨迹层

职责：

- 在感受器内部维护最近若干 tick 的对象连续性
- 进行轨迹匹配、速度估计、稳定度评估、对象感评估

输出：

- `dynamic_tracks`
- `global_motion_baseline`

### 5.4 L3：动态摘要输出层

职责：

- 只输出少量可被下游认知链消费的动态摘要对象
- 不输出整条历史轨迹

输出：

- `dynamic_motion_samples`
- `dynamic_track_summary`

---

## 6. 候选对象的正式设计

### 6.1 候选对象不是类别，而是连续结构体

每个 `shape_candidate` 建议包含：

1. 空间信息
   - `bbox_x`, `bbox_y`, `bbox_w`, `bbox_h`
   - `centroid_x`, `centroid_y`
   - `dx_from_gaze`, `dy_from_gaze`, `dr_from_gaze`

2. 规模与稠密信息
   - `area_ratio`
   - `bbox_fill`
   - `aspect_ratio`
   - `edge_density`
   - `contrast_mass`

3. 形状信息
   - `hole_count`
   - `center_void`
   - `horizontal_symmetry`
   - `vertical_symmetry`
   - `straight_likeness`
   - `curvilinear_likeness`
   - `angularity`
   - `roundness`
   - `opening_dir_x`, `opening_dir_y`
   - `opening_direction_strength`

4. 分布签名
   - `proj_h_bin`
   - `proj_v_bin`
   - `quadrant_bin`
   - `radial_bin`
   - `edge_contact_bin`

5. 极性与质量
   - `foreground_polarity`
   - `global_confidence`
   - `candidate_confidence`

### 6.2 候选对象来源

推荐采用混合式来源：

1. 默认主路径：`raw_samples` 聚合
   - 主要依赖已存在的稀疏采样证据
   - 成本低，易吃缓存

2. 聚焦增强路径：局部连通/显著区域细化
   - 仅在 `visual_focus` 或动态显著度高时启动
   - 用于提高局部对象边界稳定性

## 7. 全局运动基线的正式口径

### 7.1 为什么必须有全局运动基线

如果不先估计整幅图的大体运动，则以下情况会污染动态视觉：

1. 视焦点移动
2. 屏幕滚动
3. 摄像头/窗口整体平移
4. 整体亮度或内容统一移动

因此必须先定义：

- `global_dx`
- `global_dy`
- `global_speed`

用于表示当前 tick 的整体运动基线。

### 7.2 正式残差速度口径

每个局部样本或候选对象的运动，应以残差速度为主：

`v_residual = v_local - v_global`

这样我们真正关心的是：

- 该对象相对于整个场景是否在运动
- 其边界内外是否存在明显相对速率差

### 7.3 基线估计方法

推荐白箱、低复杂度做法：

1. 从当前 `raw_samples` 中选取高稳定点
   - `edge_strength` 高
   - `structure_discriminability` 高
   - 噪声与疲劳较低

2. 与前一 tick 或前两 tick 的稳定点粗匹配

3. 对局部位移向量使用：
   - 中位数
   - 或截尾均值

得到全局位移基线。

正式要求：

- 基线估计要稳健
- 对局部快速对象不敏感
- 不必追求像传统视觉那样极高几何精度
- 只要能为后续“相对运动”提供较稳定参考即可

---

## 8. 短时动态轨迹层的正式设计

### 8.1 感受器内部缓存

建议在 `VisionSensorV1` 内部新增：

1. `self._dynamic_shape_tracks`
2. `self._recent_shape_candidate_ring`
3. `self._global_motion_history`

这些结构都属于感受器内部缓存，不直接进入状态池。

### 8.2 轨迹窗口长度

建议默认窗口：

- `4 ~ 8 tick`

理由：

1. 足以形成短时运动趋势
2. 足以支持对象感与边界差评估
3. 不会导致缓存无限增长
4. 更适合 AP 当前 tick 级实时链路

### 8.3 轨迹对象建议字段

每条 `track` 建议包含：

1. 身份与生命周期
   - `track_id`
   - `age`
   - `last_seen_tick`
   - `miss_count`

2. 几何量
   - `centroid_x`, `centroid_y`
   - `bbox_w`, `bbox_h`

3. 动态量
   - `velocity_dx`, `velocity_dy`
   - `speed`
   - `accel_dx`, `accel_dy`
   - `size_growth`

4. 稳定性量
   - `shape_stability`
   - `motion_coherence`
   - `temporal_persistence`
   - `boundary_motion_contrast`
   - `dynamic_objectness`

5. 关系量
   - `gaze_relation_score`
   - `salience_score`

---

## 9. 轨迹匹配的正式设计

### 9.1 匹配必须是连续量，而不是硬规则

当前候选对象 `c_t` 与历史轨迹 `track_j` 的匹配，建议用统一连续得分：

`match_score =`

1. `w_shape * shape_similarity`
2. `+ w_pos * centroid_proximity`
3. `+ w_size * size_similarity`
4. `+ w_aspect * aspect_similarity`
5. `+ w_motion * residual_motion_similarity`
6. `+ w_polarity * polarity_similarity`
7. `+ w_gaze * gaze_relation_similarity`

### 9.2 匹配原则

1. 若最高得分超过阈值，则合并到对应轨迹
2. 若没有足够高的匹配，则新建轨迹
3. 若轨迹连续多 tick 未命中，则衰减并淘汰

### 9.3 轨迹更新原则

更新建议使用 EMA，而不是生硬替换：

1. `centroid = ema(old, new)`
2. `size = ema(old, new)`
3. `velocity = ema(old, new)`
4. `shape_vector = ema(old, new)`

这样可以降低随机采样带来的抖动。

## 10. 动态对象感的正式指标

### 10.1 内部运动一致性

记为：

- `motion_coherence`

含义：

- 对象内部若干局部点的残差速度是否较一致

值越高，说明越像“一个整体在动”。

### 10.2 边界运动对比

记为：

- `boundary_motion_contrast`

含义：

- 边界内外两侧的残差速度是否明显不同

值越高，说明这个边界更像真实对象边界。

### 10.3 时间持续性

记为：

- `temporal_persistence`

含义：

- 最近 `K` 个 tick 中，该对象是否能持续被追踪

### 10.4 形状稳定性

记为：

- `shape_stability`

含义：

- 多 tick 上该对象的结构特征是否大致稳定

### 10.5 动态对象感总评分

正式建议：

`dynamic_objectness =`

1. `a * motion_coherence`
2. `+ b * boundary_motion_contrast`
3. `+ c * temporal_persistence`
4. `+ d * shape_stability`

该量应作为：

1. 是否值得输出动态摘要
2. 是否值得吸引注意力
3. 是否值得触发局部采样加密

的重要依据。

## 11. 动态视觉如何帮助对象整体认知

这是本方案最重要的理论意义之一。

对象整体感不应只由静态形状决定，还应由下列因素共同决定：

1. 对象内部残差运动一致
2. 对象边界内外残差速度差明显
3. 该结构在多个 tick 内持续存在
4. 其形状虽略有变化，但仍可模糊匹配为同一对象

这意味着：

- 一个正在走动的人
- 一个在滚动的球
- 一个飘过屏幕的图标

都更容易作为“整体对象”被组织起来，而不是只被看成许多零散局部特征。

---

## 12. 与注意力和情绪链条的关系

### 12.1 对注意力的作用

以下情形应提高视觉注意力价值：

1. `dynamic_objectness` 高
2. `boundary_motion_contrast` 高
3. `motion_surprise` 高
4. `temporal_persistence` 高但把握不足

### 12.2 对“惊”的作用

以下情况可以作为视觉惊的来源：

1. 新对象突然出现
2. 旧对象突然消失
3. 速度突变
4. 方向突变
5. 边界外观稳定但相对速度异常

### 12.3 对“把握感”的作用

若同一对象连续多 tick 被稳定追踪，且结构与运动预测逐步吻合，则：

1. `grasp` 应上升
2. 对相关视觉行动的阈值可适度下降
3. 对相关预测包的承诺度可略增

---

## 13. 输出到状态池的正式口径

### 13.1 不允许整条动态历史直接入池

不得把最近 `K` tick 的所有视觉轨迹历史直接并入 `multimodal_items`。

这样会：

1. 把状态池体量打热
2. 放大重复冗余
3. 污染预测链

### 13.2 允许进入状态池的，只能是摘要对象

建议只输出少量：

1. `visual_dynamic_track_unit`
   - 表示某个稳定动态对象存在

2. `visual_motion_feature_unit`
   - 表示方向、速度、接近/远离

3. `visual_boundary_motion_unit`
   - 表示边界内外相对速度差显著

4. `visual_motion_surprise_unit`
   - 表示动态视觉中的突发变化

### 13.3 严格限流

建议每 tick 最多进入状态池：

1. `2 ~ 6` 个动态对象摘要
2. 每对象最多 `1 ~ 3` 个摘要 SA

---

## 14. 效能优化与热路径原则

### 14.1 第一原则：感受器内算完，认知链只看摘要

这是最重要的效能边界。

### 14.2 第二原则：候选数和轨迹数都要硬上限

建议：

1. 背景模式：候选 `8 ~ 24`
2. 聚焦模式：候选 `16 ~ 48`
3. 活跃轨迹数：不超过 `32 ~ 64`

### 14.3 第三原则：只做增量更新

每 tick 不回溯重算全部历史，而是：

1. 生成当前候选
2. 与活跃轨迹匹配
3. 更新 EMA
4. 淘汰低价值轨迹

### 14.4 第四原则：全局基线优先于局部复杂计算

先把 `global_motion_baseline` 做稳，再做复杂对象级差异。

因为很多伪运动都能在这一步被消掉。

### 14.5 第五原则：仅在必要时做局部细化

局部连通/边界精化应只在以下情况触发：

1. `attention_boost_active`
2. `visual_focus`
3. `dynamic_objectness` 或 `motion_surprise` 高

---

## 15. 对前端观测台的正式要求

前端应支持以下可视化：

1. 全局运动基线箭头
2. 动态对象轨迹箭头
3. 轨迹尾迹
4. 对象候选边界框
5. 边界运动对比热区
6. 动态摘要列表
7. 开关控制：
   - 原图显示
   - 仅动态层
   - 仅注视层
   - 仅重建层

这样才能帮助验收与调参，而不是只能看抽象数值。

## 16. 建议的实现顺序

### Phase A

1. 新增 `global_motion_baseline`
2. 新增内部轨迹缓存骨架
3. 新增输出字段但先不入状态池

### Phase B

1. 生成 `shape_candidates`
2. 实现轨迹匹配
3. 实现 `motion_coherence`
4. 实现 `temporal_persistence`

### Phase C

1. 实现 `boundary_motion_contrast`
2. 实现 `dynamic_objectness`
3. 与注意力聚焦采样率联动

### Phase D

1. 输出少量 `dynamic_motion_samples`
2. 严格限流进入状态池
3. 前端观测台完成动态可视化

---

## 17. 最终基线口径

动态视觉对象层的最终定位应是：

1. 它不是外挂视频算法
2. 它不是大规模历史帧全量入池
3. 它不是语义类别识别器
4. 它是：
   - 基于静态结构与短时运动的对象连续性建模层
   - 基于相对速率和边界速度差的对象整体性增强层
   - 连接“看到局部特征”和“看到一个东西在动”之间的关键桥梁

这就是后续实现时应始终守住的正式预期。
