## 1. 报告目的

---

## 2. 设计目标与哲学位置

动态视觉对象层要解决的，不是简单的“检测画面哪里变了”，而是更接近以下问题：

1. 连续多个 tick 的视觉输入中，哪些局部结构更可能属于同一个持续存在的对象。
2. 这些对象是否相对于周围场景发生了持续、可分离、可追踪的运动。
3. 这些运动信息是否可以被压缩成统一状态池可消费的少量动态摘要，而不是把所有视频历史原样倒入状态池。

因此，它不是外挂式视频识别器，也不是写死类别的传统对象检测器，而是 AP 感受器内部的一层短时加工结构。它与文本残响层的哲学位置相近：

- 近期视觉输入先进入感受器内部缓存；
- 在感受器内做连续比较、聚类和追踪；
- 只把提炼后的动态对象摘要送入统一认知链。

这一点很关键：我们当前验收通过的，不是“系统已经理解了人、车、动物这些语义类别”，而是“系统已经具备了以连续量方式形成动态对象感的基础能力”。

## 3. 当前已经落地的实现链路

当前正式实现已经从配置、感受器、运行时到观测链全部打通。

### 3.1 配置与运行时链

对应落地点：

- `track_window = 6`
- `candidate_limit_background = 12`
- `candidate_limit_focus = 28`
- `track_limit = 40`
- `summary_limit = 4`
- `match_threshold = 0.46`
- `track_forget_ticks = 3`

### 3.2 视觉感受器内部链

当前 `VisionSensorV1` 已具备以下动态视觉主链：

1. 原图分辨率偏置采样  
   基于亮度、边缘、视焦点、局部结构等因素采样局部视觉证据。

2. `shape_candidates` 候选层  
   从当前 tick 的视觉证据中聚合出有限个结构候选，而不是语义类别。

3. 全局运动基线估计  
   不再直接把“局部变化”当成对象运动，而是先估计整幅图的大体运动基线。

4. 动态轨迹维护  
   使用候选结构在近期 tick 内进行连续匹配，维护 `dynamic_tracks`。

5. 动态对象摘要输出  
   输出少量 `dynamic_motion_samples` 和 `dynamic_track_summary`，让下游统一认知链可读。

### 3.3 本次通过验收的关键修正

这次最终能够通过验收，不是偶然跑出一个漂亮样例，而是以下几处关键修正共同成立：

1. 候选结构生成从“简单间距抑制”改为“聚类式候选聚合”
2. 全局运动估计改为基于候选匹配而不是粗糙局部差分
3. 轨迹匹配时使用“上一帧位置 + 全局运动基线”的期望位置
4. `dynamic_object_count` 改为更严格的“高置信动态对象数”，而不是所有轨迹数

这四点直接决定了为什么静态场景不再大量误报、整体平移不再被误识别成满屏动态对象、单对象和多对象运动也开始具备更稳定的对象感。

## 4. 验收实验设计

正式验收脚本：

- `H:\AP原型实验第二期\scripts\run_v2_dynamic_vision_acceptance.py`

最终通过产出目录：

- `H:\AP原型实验第二期\outputs\dynamic_vision_acceptance\20260522_153538\report.md`
- `H:\AP原型实验第二期\outputs\dynamic_vision_acceptance\20260522_153538\summary.json`

本轮正式验收包含 5 个核心用例：

### A. 静态场景零假运动实验

目标：

- 连续输入同一张图时，不应凭空制造稳定的高动态对象。

### B. 整体平移去污染实验

目标：

- 区分“整幅图一起动”和“对象相对背景在动”。

### C. 单对象相对运动实验

目标：

- 在静态背景上，一个持续运动的对象应被追踪成相对稳定的同一对象。

### D. 多对象相对速率分离实验

目标：

- 多个对象同时运动时，应尽可能区分不同轨迹和不同速度模式。

### H. 规模与压力稳定性实验

目标：

- 在较长序列与较多对象维护下，轨迹数和动态摘要不应失控。

## 5. 最终验收结果

最终正式验收结果为：`5 / 5 全部通过`。

### 5.1 实验 A：静态场景零假运动

- `tail_mean_global_motion_speed = 0.0177`
- `tail_peak_dynamic_salience = 0.3035`
- `tail_mean_track_count = 16.0`

解释：

- 全局运动速度均值很低，说明静态图不会被错误地解释成整体在移动。
- 虽然内部仍维护若干轨迹，但高置信动态对象没有被稳定错误点亮。
- 这说明“轨迹存在”与“动态对象被认定”为两件不同的事，当前区分开始成立。

### 5.2 实验 B：整体平移去污染

- `tail_mean_global_motion_dx = 0.017`
- `tail_mean_global_motion_speed = 0.0211`
- `tail_mean_dynamic_object_count = 1.25`

解释：

- 系统能够测出整体平移方向；
- 但不会把整幅图都拆成大量相互独立的动态对象；
- 说明“全局运动基线先扣除，再看相对运动”的主逻辑已经生效。

### 5.3 实验 C：单对象相对运动

- `tail_peak_temporal_persistence = 1.0`
- `tail_peak_dynamic_objectness = 0.6158`
- `tail_peak_track_speed = 0.1761`

解释：

- 持续性达到 `1.0`，说明存在至少一条轨迹在尾段具有稳定连续性；
- 动态对象感超过 `0.6`，说明单对象的相对运动证据已足以被识别为高价值动态对象；
- 这代表系统已经具备“同一对象在多个 tick 中持续存在并运动”的基础建模能力。

### 5.4 实验 D：多对象相对速率分离

- `tail_dual_track_rows = 8`
- `tail_mean_speed_divergence = 0.0564`

解释：

- 在尾段 8 个观测点上，系统都能稳定给出至少两条可读轨迹；
- 两条轨迹之间的速度差不是 0，说明没有完全粘成一个对象；
- 这证明“相对速率差帮助对象边界分化”的思路在当前原型里开始成立。

### 5.5 实验 H：规模与压力稳定性

- `tail_mean_track_count = 16.0`
- `tail_max_track_count = 16.0`
- `tail_mean_dynamic_salience = 0.3247`
- `overall_mean_global_motion_speed = 0.0197`

解释：

- 在这一轮压力测试中，轨迹规模保持稳定，没有出现失控膨胀；
- 动态显著度也没有在压力下异常炸高；
- 这说明当前对象层在小规模到中等规模序列中已经具备工程可运行性。

## 6. 当前结果能够证明什么

### 6.1 已经可以证明的部分

当前实验结果已经可以较严谨地支持以下结论：

1. **动态视觉对象层已经真实落地，不是纸面设计。**  
   它已经从配置到运行时再到验收脚本形成了完整闭环。

2. **系统已经能区分“整体运动基线”和“对象相对运动”。**  
   这点是动态视觉能否成立的基础。如果没有这点，后续所有运动对象感、注意力聚焦、OCR-like 连续视觉认知都会被满屏假运动污染。

3. **系统已经具备“对象级短时连续性”的基础。**  
   当前不是只会看单帧差分，而是已经能把一段时间内相近结构视作一个持续对象。

4. **系统已经初步具备“多对象相对运动分化”的能力。**  
   这说明动态视觉不再只是“检测变化区域”，而开始具备把不同变化模式拆成不同对象的能力。

5. **当前实现已经具备进入下一阶段联动实验的资格。**  
   也就是用户之前提出的那条路线：  
   注意力拉动采样率、运动对象驱动惊/聚焦、再往 OCR-like 连续视觉认知那边接。

### 6.2 这些结果的理论意义

这意味着 AP 第二期当前已经不只是“静态图像特征进入状态池”，而是开始拥有：

- 时间连续性；
- 对象持续性；
- 相对运动结构；
- 动态摘要可入统一认知链。

这几件事合起来，才是更拟人的“看见一个东西在动”的雏形，而不是普通意义上的像素变化统计。

## 7. 当前结果还不能证明什么

这部分必须说清楚，否则会高估现状。

### 7.1 还不能证明语义级视频理解

当前通过的，是“动态对象层”验收，不是“动态语义理解”验收。

也就是说，目前还不能仅凭这组实验就声称系统已经能稳定理解：

- 这是一个人；
- 这是一个手在挥动；
- 这是某种复杂行为意图。

当前证明的是“对象持续性与相对运动结构”这一层，而不是高层语义类别。

### 7.2 还不能证明与注意力、惊、把握感的端到端联动已经完全成立

虽然运行时里已经存在：

- 注意力 boost；
- 惊 / 正确感 / 把握相关的内在反馈通道；
- 动态视觉摘要进入多模态输入链；

但本轮正式验收并没有把“动态视觉 -> 惊 -> 聚焦 -> 重新采样 -> 认知恢复”作为主验收对象。

所以这一段目前属于“具备接口基础，但尚未完成端到端正式证明”。

### 7.3 还不能证明 OCR-like 连续视觉认知已经靠动态视觉打通

当前动态视觉验收并没有直接测试：

- 连续多个 tick 观察手写体；
- 因运动、注视和采样率变化而提升识别稳定性；
- 再通过 AP 的统一记忆链召回对应文本。

因此，OCR-like 连续视觉认知还需要单独的联动实验来证明。

### 7.4 还不能证明真实视频大规模吞吐下的最终性能边界

当前压力实验说明“小到中等规模序列下结构稳定”，但还没有覆盖：

- 数百到上千 tick 连续视频流；
- 更高分辨率；
- 更复杂背景；
- 更密集动态对象；
- 与完整状态池记忆链共同运行时的全链热点。

所以性能边界还要继续测。

## 8. 为什么这轮验收是可信的

这轮结果可信，不是因为它“看起来不错”，而是因为它满足了下面几件事：

1. 不是单个截图演示，而是成组验收脚本；
2. 不是只测动态对象，而是同时测静态、整体平移、单对象、多对象、压力稳定性；
3. 不是只看前端观感，而是看内部连续量指标；
4. 不是只要出现轨迹就算成功，而是明确区分了轨迹、动态对象感和高置信对象计数；
5. 最终结果有固定产出目录，可以复查、复跑、复核。

因此，这是一轮“足以作为下一阶段研发基线”的工程验收，而不是一次灵感性成功。

---

## 9. 当前主要局限与风险

### 9.1 局限

1. 仍偏二维平面动态视觉，尚未接入三维深度或真实双目定位。
2. 候选结构仍主要依赖图像结构连续量，尚未形成更强的跨 tick 语义对象抽象。
3. 长时视频下，轨迹遗忘、对象重现、遮挡恢复等还没有做系统性验收。
4. 与行动链、情绪链的联动虽然已有接口，但还缺正式联合实验报告。

### 9.2 风险

1. 如果后续为了追求“看起来更灵”，随意放宽匹配阈值或放松高置信对象判定，静态误报和整体平移污染很可能回潮。
2. 如果后续把大量动态原始历史直接倒入状态池，可能破坏当前“感受器内部加工，状态池只接摘要”的哲学边界。
3. 如果后续联动实验里注意力增益过强，可能把动态视觉变成单纯的硬聚焦脚本，需要谨慎调参。

## 10. 面向下一阶段的结论

当前可以给出的正式结论是：

**V2 原型的动态视觉对象层已经通过基础验收。**

它现在已经能够：

- 在静态场景下抑制假运动；
- 在整体平移下建立全局运动基线；
- 在单对象运动下维持持续轨迹；
- 在多对象运动下分化不同速度模式；
- 在当前小到中等规模压力下保持结构稳定。

因此，下一阶段可以合理进入“动态视觉联动实验”，重点验证以下链条：

1. 注意力是否能根据运动对象或惊信号拉动视觉采样率；
2. 运动对象是否能更稳定地驱动惊、聚焦、继续观察；
3. 连续多个 tick 的视觉聚焦是否能改善 OCR-like 连续视觉认知；
4. 动态视觉摘要进入统一状态池后，是否能更好地触发多模态记忆召回与认知恢复。
