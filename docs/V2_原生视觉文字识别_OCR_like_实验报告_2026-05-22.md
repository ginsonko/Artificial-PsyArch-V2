# V2 原生视觉文字识别（OCR-like）实验报告

## 1. 实验目标

本实验的目标，不是验证传统 OCR 算法，而是验证 AP V2 原型是否能够仅依靠自身的视觉稀疏采样、状态池、记忆召回、预测与奖励塑形链路，形成一种原生的图像文字识别雏形。

更具体地说，本实验想回答三个问题：

1. 当一张手写数字图像与一个文本标签持续共现时，系统能否把该图像特征与对应文本标签绑定起来。
2. 训练完成后，仅再次输入图像而不给文本，系统能否从记忆中召回正确文本。
3. 在先看过另一张图像、存在旧上下文干扰的情况下，系统能否经过若干个连续 observation tick，从旧认知切换到新认知。

## 2. 实验对象与刺激材料

本轮实验使用两张程序生成的手写风格数字图像：

- `digit_3`：图像内容为数字 `3`，配套文本标签为 `three`
- `digit_8`：图像内容为数字 `8`，配套文本标签为 `eight`

对应图像文件：

- [digit_3.png](H:\AP原型实验第二期\outputs\vision_ocr_probe\20260521_233420\generated_images\digit_3.png)
- [digit_8.png](H:\AP原型实验第二期\outputs\vision_ocr_probe\20260521_233420\generated_images\digit_8.png)

## 3. 系统链路与判定口径

实验中，图像不会进入传统 OCR 模块，也不会先做字符分类器，而是进入 AP V2 的原生链路：

1. 图像进入视觉感受器，生成原图分辨率上的稀疏采样特征与全局结构特征。
2. 文本标签与视觉特征同时进入状态池。
3. 系统通过 `Bn` 召回历史记忆，通过 `C*` 形成当前综合预测包。
4. 由更新后的状态池再读出当前 `A_focus`。
5. 训练阶段同步注入奖励信号，使“图像-文本正确绑定”得到长期塑形。

本实验主要使用四个观察层级来判断识别是否成立：

- `BN_top`：最强候选记忆的文本标签
- `C*_top`：当前综合预测包中最强文本标签
- `State_top_text`：状态池顶层文本能量峰值对应的标签
- `A_focus`：当前显意识焦点中是否已经出现目标文本或目标对象路径

其中：

- `BN_top` 正确，说明记忆召回已经开始对上
- `C*_top` 正确，说明当前现状整合后的认知判断已经对上
- `State_top_text` 正确，说明状态池主导波峰已经切换到正确对象
- `A_focus` 命中，说明显意识主焦点已经开始选中正确路径

## 4. 训练设计

训练阶段配置如下：

- 训练轮次：12 epoch
- 每个 epoch：交替输入 `digit_3 + three` 与 `digit_8 + eight`
- 视觉采样主预算：raw = 256
- 记忆写入预算：memory_write = 16
- 焦点记忆预算：focus = 8
- 稳定空 tick：10
- 每个正确图像-文本共现 tick 注入奖励：`reward = 1.0`

训练摘要来自 [summary.json](H:\AP原型实验第二期\outputs\vision_ocr_probe\20260521_233420\summary.json)：

- raw budget = 256
- patch budget = 16
- focus budget = 8
- stabilize ticks = 10
- reward = 1.0

## 5. 预期

如果 AP V2 的原生视觉识别链路成立，那么预期会出现以下现象：

1. 在训练若干轮之后，面对 `digit_3`，`BN_top` 和 `C*_top` 会稳定偏向 `three`。
2. 面对 `digit_8`，`BN_top` 和 `C*_top` 会稳定偏向 `eight`。
3. 即便只给图像、不再给文本标签，系统也能从历史记忆中召回正确文本。
4. 在存在旧图像上下文干扰时，系统不一定第一反应就正确，但连续观察数个 tick 后，应能够逐步完成认知切换。

## 6. 主实验结果

### 6.1 冷启动探测结果

训练完成并经历稳定空 tick 后，对两张图像分别做冷探测，结果如下：

- `digit_3`：`BN_top = three`，`C*_top = three`，严格判定成功
- `digit_8`：`BN_top = eight`，`C*_top = eight`，严格判定成功

从 [report.md](H:\AP原型实验第二期\outputs\vision_ocr_probe\20260521_233420\report.md) 可见：

- `digit_3` 的 `C* margin = 115.1551`
- `digit_8` 的 `C* margin = 46.4996`

这说明，在没有再提供文本标签的情况下，仅凭图像输入，系统已经能够把对应文本标签从记忆中拉起来，而且在 `C*` 层形成明显优势。

### 6.2 接受门槛扫描结果

在训练后、以 `raw = 256` 的设置进行多轮 observation tick 扫描时：

- `digit_3` 达到严格成功
- `digit_8` 达到严格成功
- 该采样率下首次达到 `strict_accuracy = 1.0` 所需 observation ticks = 1

这意味着：在当前这组较小任务上，256 级 raw 视觉采样已经足以建立可工作的 OCR-like 基础能力。

## 7. 强干扰上下文切换实验

为了验证系统不是只会“静态认图”，还需要验证在旧上下文干扰下，它能否经过连续观察完成认知切换。

因此设计了一个更强的协议：

1. 先完成 `3/8` 配对训练
2. 稳定若干空 tick
3. 连续观察 4 个 tick 的 `digit_3`
4. 不清池，立刻切换到 `digit_8`
5. 连续观察 `digit_8` 多个 tick，记录每一层何时翻转

### 7.1 关键结果

在该协议下，`digit_8` 的切换结果为：

- `BN_top` 第 2 个 observation tick 翻到 `eight`
- `C*_top` 第 3 个 observation tick 翻到 `eight`
- `A_focus` 第 3 个 observation tick 已命中 `eight` 路径
- `State_top_text` 第 6 个 observation tick 翻到 `eight`

### 7.3 解释

这个结果很有价值，因为它说明 AP V2 的视觉识别不是单纯的一次性静态判别，而更像一种跨 tick 的认知更新过程：

- 初始时，旧的 `three` 仍然占据主导，这很像人类在刚切换对象时还保留前一认知痕迹
- 随着连续 observation tick 输入，`BN` 先翻转，说明记忆召回先被纠正
- 接着 `C*` 翻转，说明当前整合判断对上
- 再之后 `State_top_text` 才翻转，说明主导波峰完成切换

也就是说，这个系统允许“第一反应不一定对”，但在连续观察中能逐步修正自己，这比要求首 tick 必须正确，更符合拟人化认知哲学。

## 8. 实验意义

本轮实验能够支持以下阶段性结论：

1. AP V2 已具备一种原生的 OCR-like 基础能力。
2. 该能力不是来自传统 OCR 管线，而是来自视觉采样特征、状态池、记忆召回、预测与奖励塑形的统一链路。
3. 图像与文本标签之间已经可以建立可召回的绑定关系。
4. 在旧上下文干扰存在时，系统可以通过连续 observation tick 逐步切换认知对象。

如果把它翻译成更直观的话，就是：

- 它已经不只是“看到一堆像素”
- 而是已经开始能把“这个视觉对象”和“这个词”联系起来
- 并且在认知切换时表现出一种跨 tick 的逐步收敛过程

## 9. 当前边界

本实验证明的是“原生视觉文字识别雏形已经成立”，但还不能直接推出以下更强结论：

1. 不能直接等价于成熟通用 OCR。
2. 还不能证明跨字体、跨噪声、跨布局、跨复杂场景下的广泛泛化。
3. 还不能证明对真实自然图片中的任意文字都能稳定识别。
4. 当前更接近“带奖励配对训练后，图像与文本标签可以形成可召回绑定”。

因此它是一个非常重要的能力起点，但还不是终点。

## 10. 阶段性总结

如果只看“有没有这种能力”，答案已经是肯定的。

这轮实验最值得向同事分享的点，不是它已经达到传统 OCR 产品级水平，而是：

> **AP V2 这种统一状态池 + 记忆召回 + 预测包 + 奖励塑形的架构，已经能够不依赖传统 OCR 模块，直接长出一种原生的视觉文字识别能力雏形。**

而且这个能力不是死板分类器式的单步输出，而是具有：

- 连续观察
- 旧上下文干扰
- 逐步翻转
- 记忆驱动召回

这些更接近认知系统的行为特征。
