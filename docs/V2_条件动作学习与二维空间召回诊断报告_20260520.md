# AP 二期 V2 条件动作学习与二维空间召回诊断报告

日期：2026-05-20

## 1. 结论先说

这次排查后的核心结论有两条：

1. 你关于“奖励提升行动冲动、惩罚抑制行动冲动、全局正负情绪再通过阈值调节长期塑形”的理论方向本身是成立的，而且当前 V2 里已经实现了其中一部分，但还没有完整实现成一个足够强的条件动作学习系统。
2. 当前 V2 的空间链条已经不是 0。视觉 patch 的二维坐标已经进入了记忆写入、向量特征和时空邻域索引，但它现在仍然偏弱，主要问题是“空间信息被压得太粗”，还没有成为一级召回的强条件，因此还不能严格证明空间召回能力。

换句话说：

- 奖励/惩罚信号链已经通了。
- 违和感/正确感/期待/压力链已经通了。
- 但“强条件动作反射”还没有被严格证明。
- 二维空间信息已经进入系统，但目前更像弱辅助通道，还不是主导召回通道。

## 2. 这次用于判断的直接证据

### 2.1 条件动作学习补充探针的当前结果

文件：

- `outputs\conditioned_learning_probe\conditioned_learning_probe_report.json`

关键结果：

- `baseline_probe_accuracy = 0.5`
- `trained_probe_accuracy = 0.5`
- `passed_conditioned_action_learning = false`
- `passed_punishment_pressure_signal = true`
- `passed_reward_expectation_signal = true`

这说明：

1. 奖励之后，期待确实会上升。
2. 惩罚之后，压力确实会上升。
3. 但动作选择精度没有从训练前 0.5 提升到训练后更高水平，所以“奖惩已进入链条”不等于“条件动作习惯已经形成”。

### 2.2 当前动作学习实现的关键位置

文件：

- `core\action_learning_v2.py`
- `core\runtime_v2.py`
- `core\teacher_layer_v1.py`
- `scripts\run_v2_conditioned_learning_probe.py`

动作学习的直接实现要点：

1. `ActionLearningV2.score_action_drives(...)` 当前是：
   - 基础 drive
   - 加上全局动作 bias
   - 再加上上下文动作 bias
2. `ActionLearningV2.record_feedback(...)` 当前把
   - `expectation`
   - `correctness`
   - `pressure`
   - `dissonance`
   - 外部 reward / punishment
   合成为一个单一 `valence`
3. 这个 `valence` 再回写到：
   - 被选中的动作全局 bias
   - 被选中的动作在当前上下文下的 context bias

### 2.3 当前空间链条实现的关键位置

文件：

- `sensors\vision_sensor_v1.py`
- `memory\memory_store_v2.py`
- `memory\embedding_v2.py`
- `memory\spacetime_index_v2.py`

当前已经存在的空间链条：

1. 视觉 patch 自带 `row/col/cx/cy`
2. 写记忆时 `_infer_spacetime(...)` 会把 item 的 `cx/cy` 聚合成记忆的 `x/y`
3. `HashEmbeddingV2.build_memory_vector(...)` 会把 `spacetime` 量化成 `sp::x::*` / `sp::y::*` token
4. `SpacetimeIndexV2` 会按 `x/y/z` 桶建立空间索引，并在二级邻域召回里计算 `space_bonus`

所以当前空间信息不是只放日志里，而是已经实打实进入：

- 记忆向量
- 时空索引
- 二级预测邻域评分

## 3. 为什么海豚训练类实验还没有达到理论预期

## 3.1 第一层判断：不是理论错了，而是实现还不完整

你的理论主张可以拆成四层：

1. 奖励应提升相关行动倾向
2. 惩罚应压低相关行动倾向
3. 正向全局情绪应降低行动门槛
4. 负向全局情绪应抬高行动门槛

其中当前已经明确落实的是：

- 第 1 层：部分落实
- 第 2 层：部分落实
- 第 3 层：没有作为独立全局阈值系统完整落实
- 第 4 层：没有作为独立全局阈值系统完整落实

所以更准确地说，当前 V2 不是“理论不成立”，而是“只做了局部动作偏置学习，还没做完整的情绪-阈值-行动系统”。

---

## 3.2 当前最主要的 5 个断点

### 断点 A：现在学到的是 bias，不是完整的行动阈值系统

当前 `action_learning_v2.py` 的核心是：

- 为动作学全局 bias
- 为动作学上下文 bias

但没有独立维护一个类似：

`action_threshold = base + cortisol_like - dopamine_like`

的全局门槛变量。

也就是说，现在的期待/压力主要用于：

1. 参与反馈 valence
2. 参与规则层生成 action drive

但没有变成一个持续存在的“做不做动作”的系统级门槛。

这会导致：

- 奖励/惩罚能够改 bias
- 但不能稳定地改变整轮动作竞争的出手门槛

所以它更像“动作偏好校正”，还不是“习惯性冲动塑形”。

### 断点 B：当前反馈只更新被选中的动作，不更新竞争失败的正确动作

`record_feedback(...)` 只对 `selected_actions` 做更新。

这意味着：

1. 选错了动作时，会惩罚错误动作
2. 但不会同时显式提升“本该被选中的正确动作”

这会让学习非常慢，因为系统没有学到：

- “错的是谁”
- “应该替代它的是谁”

而只学到：

- “这次选中的那个家伙不太好”

这类更新对于二选一条件反射任务太弱。

### 断点 C：当前上下文键太碎，而且存在跨命令污染

当前上下文键来源于：

- `text::完整文本`
- `focus_text::焦点串`
- `focus::单个焦点 SA`
- `unit::单个 query unit`
- `short_term::近期短期记忆`

这在理论上是丰富的，但在条件动作学习里有个实际问题：

`打开记事本` 和 `打开计算器` 共享很多低层 token，例如：

- 打
- 开
- 打开

于是会发生：

1. 对“记事本”学到的正负偏置，会通过共享 token 外溢到“计算器”
2. 对“press_key”在某上下文学到的负偏置，会被泛化到别的共享上下文
3. 对“type_text”在某上下文学到的正偏置，也会被泛化出去

这会造成你现在看到的现象：

- 系统确实学到了一些 context bias
- 但这些 bias 不是干净地按任务对象分离的
- 而是掺了共享 token 带来的串扰

### 断点 D：当前 probe 数据集训练方式本身也在稀释学习

`run_v2_conditioned_learning_probe.py` 的训练步骤中，同一个命令上下文会交替使用：

- `wrong_bias`
- `correct_bias`

这意味着训练时不是单纯让系统从统一环境中学习“正确动作应该胜出”，而是人为在底层 drive 上不断切换偏置方向。

这样做适合“压测能否扭转错误先验”，但不适合作为第一性条件反射验证。

原因是：

1. 学习目标不够稳定
2. 错误 base drive 和正确 base drive 在同一训练序列里交替出现
3. 训练信号不再纯粹反映“命令 -> 正确动作”的条件映射

所以这个 probe 更像“高难 stress test”，不是最干净的首轮证明实验。

### 断点 E：当前系统还没有显式的对比式学习更新

条件动作学习最有效的形式通常不是单点加减，而是 margin 学习：

- 正确动作上升
- 错误动作下降
- 两者差距被拉大

当前实现更接近：

- 只改被选中的那个动作

而不是：

- 同上下文下同时拉开正确动作和错误动作的差距

这会让模型很容易停在：

- “我知道有点不对”
- “但我还没有足够证据把另一个动作真正推上去”

## 3.3 当前实验为什么会出现“情绪链对了，但动作学习没站稳”

这是这次最重要的口径统一点。

现在系统已经证明：

1. 奖励会提高期待
2. 惩罚会提高压力
3. 违和感/正确感会随着预测命中与否变化

但这只能证明：

- 情绪和奖惩信号链是通的

还不能自动推出：

- 它们已经足够强地塑造成了稳定动作习惯

中间还缺了一层：

- 情绪和奖惩如何持续、选择性地改变动作门槛与动作间 margin

这正是当前 V2 尚未完全落地的部分。

## 4. 条件动作学习的改进方案

## 4.1 最高优先级：补上“全局行动阈值层”

建议新增长期状态：

- `dopamine_like`
- `cortisol_like`
- `global_action_threshold`

建议关系：

1. 奖励、期待、正确感抬升 `dopamine_like`
2. 惩罚、压力、违和感抬升 `cortisol_like`
3. 行动是否执行由
   - `drive`
   - `global_action_threshold`
   - `action_family_threshold`
   共同决定

建议最小公式：

`threshold = base + a * pressure + b * dissonance - c * expectation - d * correctness - e * recent_reward`

这一步是把你的“多巴胺/皮质醇哲学”真正变成工程结构的关键。

## 4.2 第二优先级：把反馈更新改成对比式

在存在 `expected_action` 的训练任务里，建议把反馈更新改为：

1. 若选对：
   - 提升 `selected_action`
   - 轻微压低同上下文下的竞争动作
2. 若选错：
   - 压低 `selected_action`
   - 同时提升 `expected_action`

这样才能真正形成：

- `drive(correct) - drive(wrong)` 的 margin 扩大

而不是只有单点惩罚。

## 4.3 第三优先级：上下文键分层，避免串扰

建议把上下文键拆成三层：

1. 强键：
   - `command_id::open_notepad`
   - `command_id::open_calc`
2. 中键：
   - `text::打开记事本`
   - `text::打开计算器`
3. 弱键：
   - `unit::打`
   - `unit::开`
   - `focus::开`

学习时：

- 强键权重大
- 中键次之
- 弱键只做泛化辅助

这样共享 token 就不会把任务学坏。

## 4.4 第四优先级：重做条件动作学习实验集

建议分成三档：

### A. 干净验证集

目的：先证明命令到动作的条件映射能学会。

设计：

- 固定上下文
- 中性 base drive
- 大量重复
- 奖励只给正确动作
- 惩罚只给错误动作

通过标准：

- 训练前接近随机或偏错
- 训练后 > 90%

### B. 错先验扭转集

目的：证明它能推翻错误初始偏置。

设计：

- 先人为给错动作更高初始 drive
- 再用干净反馈持续训练

通过标准：

- 正确动作 margin 从负转正

### C. 干扰鲁棒集

目的：证明共享 token、空 tick、其它命令插入后仍能保持习惯。

设计：

- 插入 `打开浏览器`、`等待`、空 tick
- 控制上下文切换

### D. 训练量建议

当前 8 轮重复对条件反射仍偏少。

建议：

- 干净验证：50-200 次有效暴露
- 错先验扭转：100-300 次有效暴露

## 4.5 第五优先级：加入探索机制

如果一个动作略微领先，系统可能长期只重复它，另一个动作得不到足够正反馈。

建议：

1. 训练模式下加入小概率探索
2. 或者前若干轮强制 balanced exposure

否则错误动作一旦先赢，就可能形成“越选越学、越学越选”的坏闭环。

## 5. 二维空间召回的当前现状判断

## 5.1 当前已经做到了什么

目前视觉 patch 的二维信息已经进入系统：

1. `VisionSensorV1` 为每个 patch 记录 `cx/cy`
2. 记忆写入时 `_infer_spacetime(...)` 会生成 `x/y`
3. embedding 会写入 `sp::x::*` / `sp::y::*`
4. 时空邻域索引会使用 `space_bonus`

所以当前不能说“空间通道还没接上”。

更准确的说法是：

> 空间通道已经接上，但还只是粗粒度、弱权重、以记忆中心点为主的版本。

## 5.2 当前为什么还不足以严格证明空间召回

主要有四个原因：

### 原因 A：空间被压缩成单个中心点

`_infer_spacetime(...)` 目前基本是对所有 patch 的 `x/y` 求平均。

问题是：

- 左右两个目标交换位置，但整体平均值可能接近
- 形状信息、分布信息、边界信息都丢了

所以它保留的是“这个记忆大概在图上哪里”，不是“场景内部对象的空间结构”。

### 原因 B：当前 query 侧没有显式空间向量

`build_query_vector(...)` 目前主要吃：

- label
- query_units
- recent_focus_units

没有显式输入：

- 当前视觉 query 的 `x/y`
- 相对焦点坐标
- 占据区域信息

这意味着：

- 一级召回 `Bn` 还不是“按当前空间现状直接查”
- 空间主要在记忆写入侧和二级邻域侧发挥作用

### 原因 C：没有相对焦点坐标通道

现在有 `gaze_bonus`，但它只是注意力加成，不等于显式空间关系。

缺少的是真正的：

- `rel_x = cx - gaze_x`
- `rel_y = cy - gaze_y`

以及其量化桶。

这会影响“相对于当前视焦点”的空间记忆。

### 原因 D：没有保存空间分布签名

除了均值坐标外，建议还需要：

- 左/中/右
- 上/中/下
- bounding box
- 能量加权中心
- 最强 patch 位置
- 多 patch 占据图

否则很难区分：

- “苹果在左、香蕉在右”
- “苹果在右、香蕉在左”

## 6. 二维空间召回的推荐方案

## 6.1 先明确当前阶段要证明什么

当前阶段不要试图证明：

- 真正三维空间认知
- 真实世界绝对坐标
- 图像级完整重建

当前阶段应该证明的是：

1. 图像平面二维绝对位置能够进入记忆召回
2. 相对视焦点位置能够进入记忆召回
3. 同一目标在不同位置时，召回结果会有系统性差异
4. 文本/视觉可以围绕二维位置发生绑定

这已经很有价值。

## 6.2 建议增加两套空间通道

### A. 图像平面“绝对”坐标

每个 patch / 目标加入：

- `abs_x`
- `abs_y`

可量化成：

- 左 / 中 / 右
- 上 / 中 / 下
- 更细桶如 8x8

### B. 相对焦点坐标

每个 patch / 目标加入：

- `rel_x = abs_x - gaze_x`
- `rel_y = abs_y - gaze_y`

可量化成：

- 焦点左侧 / 焦点右侧
- 焦点上方 / 焦点下方
- 距焦点近 / 中 / 远

这两套都应该进入：

1. item attributes
2. 记忆 spacetime
3. embedding token
4. 一级 query vector
5. 时空邻域索引

## 6.3 建议把空间表达从“中心点”升级为“空间签名”

记忆级空间签名建议至少包含：

1. `center_x / center_y`
2. `focus_x / focus_y`
3. `spread_x / spread_y`
4. `bbox_min_x / bbox_max_x / bbox_min_y / bbox_max_y`
5. `occupied_quadrants`
6. `top_patch_positions`

其中最重要的是前三个：

- 中心
- 相对焦点
- 扩散范围

## 6.4 一级召回也要显式吃空间 query

这是最关键的改动之一。

现在一级召回主要看：

- label overlap
- unit overlap
- bigram overlap
- vector similarity

建议加入 query 侧空间 token，例如：

- `q_abs_x::2`
- `q_abs_y::7`
- `q_rel_x::left_near`
- `q_rel_y::below_mid`
- `q_occ::left_top`

这样才能做到：

- “同样都是苹果，但左边苹果先召回左边相关记忆”

## 6.5 二级时空邻域继续保留，但改成更像“空间延展”

当前 `SpacetimeIndexV2` 已经可以保留，继续作为：

- 时间连续性
- 空间邻近性

的二级扩展层。

但它不应承担全部空间辨别责任。

更合理的职责分工是：

1. 一级召回：当前现状的主匹配
2. 二级时空邻域：围绕已命中的记忆做时空延展

## 7. 二维空间召回的实验设计

## 7.1 实验目标

证明：

1. 空间位置不同，召回不同
2. 相对焦点不同，召回不同
3. 文本与视觉可围绕位置绑定

## 7.2 实验 A：绝对二维位置召回

训练数据：

- 图像 1：苹果在左，配文本“苹果在左边”
- 图像 2：苹果在右，配文本“苹果在右边”

训练若干轮后测试：

- 只输入苹果在左的图像
- 看是否优先召回“左边”相关文本或左侧记忆

指标：

- left-condition 命中率
- right-condition 命中率
- 左右互串率

## 7.3 实验 B：相对焦点位置召回

设计：

- 同一张图，不同 tick 把 gaze 移到不同位置
- 同一目标相对于 gaze 一会在左、一会在右

测试：

- 看召回是否更偏向“焦点左侧目标”还是“焦点右侧目标”

这能证明：

- 相对坐标不是空谈，而是可影响记忆匹配

## 7.4 实验 C：多模态空间绑定

训练：

- 图片：铃铛在上方
- 文本：“铃铛在上面”
- 音频：固定铃声片段

测试：

1. 只给图片，看是否召回“上面”和铃声
2. 只给文本“上面的铃铛”，看是否召回上方视觉片段

这可以证明：

- 多模态不仅能共现绑定
- 还能围绕空间位置信息共现绑定

1. 已实现完整类人操作性条件反射

## 9. 最推荐的下一步顺序

### 第一步：先把条件动作学习做干净

顺序：

1. 增加全局动作阈值层
2. 增加对比式更新
3. 重做干净验证 probe
4. 再做错先验扭转集

原因：

- 这是最直接关乎“学习能力是否成立”的证据
- 也是你最关心的海豚训练哲学落点

### 第二步：补二维空间 query 通道

顺序：

1. 加 `abs_x/abs_y`
2. 加 `rel_x/rel_y`
3. 升级空间签名
4. 把空间 token 纳入一级 query vector
5. 跑空间召回实验

### 第三步：再做多模态空间绑定实验

这样实验链条会很漂亮：

1. 文本连续性
2. 奖惩塑形
3. 二维空间召回
4. 空间条件下的多模态绑定

这四个放在一起，V2 的辨识度会明显高很多。

---

## 10. 最终判断

如果一句话总结这次诊断：

当前 V2 在“奖惩与认知感受进入统一主链”这件事上已经成功，但在“把这些信号变成稳定动作习惯”这件事上，还差一个真正的阈值层、对比式学习层和更干净的训练协议；而二维空间方面，视觉坐标已经进入系统，但仍停留在粗粒度中心点阶段，下一步应该把绝对位置、相对焦点位置和空间分布签名一起做进一级召回。

所以这不是“哪里完全错了”，而是：

- 动作学习：主哲学对，工程层缺一段
- 空间召回：主链已接上，但信息表达还太粗

这两个问题都属于可以继续补强、而且补强后很有希望形成论文级亮点的类型。
