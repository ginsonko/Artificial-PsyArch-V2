# AP 二期 V2 节奏感详细设计方案

日期：2026-05-23

## 1. 设计定位

节奏感不是新的独立哲学体系。

它应被定义为：

1. 时间感受的高阶模式
2. 重复命中事件在短期内形成的相位锁定结构
3. 疲劳、恢复、期待、把握共同作用后形成的主体内在感觉

也就是说：

> 节奏感 = 重复事件的时间结构 + 预测相位 + 疲劳恢复平衡

而不是：

1. 简单的重复计数
2. 硬编码四拍、八拍
3. 纯粹的外部时间桶命中

## 2. 与人类哲学最接近的核心判断

最符合人类哲学的方案，不是“只要重复就有节奏感”，而是：

1. 某类事件在短时间内重复出现
2. 它们的时间间隔比较稳定
3. 主体开始预测下一次何时会来
4. 下一次事件到来时，恰好落在系统对该类事件的疲劳已部分恢复、但又还保持连续感的区间内
5. 于是既不糊成连续刺激，也不散成互不相关的孤立事件
6. 从而形成“拍点”“律动”“顺”的感觉

所以节奏感真正依赖的是：

1. 重复
2. 周期稳定
3. 预测成功
4. 疲劳恢复曲线与重复间隔的匹配

没有疲劳，系统只会单调地越来越熟。
只有疲劳，没有恢复，系统只会越来越麻木。
只有“疲劳-恢复-再命中”的循环，才更像节奏。

## 3. 外部层与主观层

### 3.1 外部描述层

节奏通道首先描述的是一类事件流的重复结构：

1. 事件族 `event_family`
2. 周期 `tau`
3. 相位 `phase`
4. 稳定度 `regularity`
5. 近期重复次数 `repeat_count`
6. 振幅或显著度 `salience`

这里的事件不局限于音频。

它可以来自：

1. 音频脉冲
2. 视觉运动对象的周期变化
3. 文本中重复结构
4. 行动-反馈循环
5. 内部认知对象的重复回想

### 3.2 主观感受层

建议第一版只引入两个节奏专属 feeling：

1. `rhythmfelt::pulse`
   - 我感到“这里在重复，而且有拍点”
2. `rhythmfelt::phase_expectation`
   - 我感到“下一拍快该来了”

而以下结果尽量复用现有通道，不重复造轮子：

1. 预测命中后的愉悦 -> 继续走 `expectation / correctness / grasp / reward`
2. 预测落空后的紧张和错配 -> 继续走 `surprise / dissonance / punishment`

这样最统一，也最优雅。

## 4. 事件族的形成

节奏感不能依赖硬编码“鼓点”“脚步”“文字”。

建议做法是：

1. 每个 tick 从当前高能量、较高 commitment 的对象里抽取一组事件候选
2. 用现有向量相似度 / 通道相似度把它们归入若干 `event_family`
3. 只有相似度超过阈值的，才视为同一类重复事件

也就是说，节奏跟踪的对象不是具体标签，而是“相似事件族”。

## 5. 节奏结构的形成

对每个 `event_family`，维护一个短窗口历史：

1. 命中时刻 `t_i`
2. 命中强度 `a_i`
3. 当时的 attention / commitment / reality support

然后计算相邻时间差：

`delta_i = t_i - t_(i-1)`

再对这些时间差做连续聚合，得到主周期假设 `tau`。

建议第一版不用复杂 FFT，而用更贴近现有工程的连续聚类法：

1. 收集最近 `N=4~8` 个间隔
2. 用每个 `delta_i` 做种子
3. 按距离核函数聚合
4. 选出质量最大的主峰作为 `tau`

可记为：

`RhythmField(d) = sum_i w_i * exp(-(d - delta_i)^2 / (2*sigma_i^2))`

主峰位置就是主周期 `tau`。

## 6. 节奏强度不应只看重复次数

建议把节奏强度定义为：

`rhythmicity = regularity * recurrence * salience_support`

其中：

1. `regularity`
   - 时间差是否集中在同一段范围
2. `recurrence`
   - 最近窗口内是否确实重复了多次
3. `salience_support`
   - 这些命中是否真的是系统注意到的对象，而不是底层噪声

这样可以避免“重复噪声也被误判成节奏”。

## 7. 疲劳与恢复为何是核心

这是本方案最关键的部分。

### 7.1 人类式解释

如果同类事件来得太快：

1. 疲劳还没恢复
2. 主体会更麻
3. 重复会糊成连续刺激
4. 节拍感下降

如果同类事件来得太慢：

1. 连续性断了
2. 主体不会把它们视为同一个流
3. 节奏感也下降

只有当周期 `tau` 落在一个“恢复刚好合适”的区间时：

1. 事件仍然彼此相关
2. 上一次命中留下的痕迹还没消失
3. 但疲劳又已部分恢复
4. 下一次命中会更清楚、更可预期

这才最像人类的“节奏感”。

### 7.2 数学表达

对参与该节奏的相关通道，设一个综合疲劳恢复时间尺度 `tau_recover`。

定义恢复匹配度：

`recovery_match = exp(-((tau - tau_recover)^2) / (2 * sigma_recover^2))`

于是节奏愉悦或“律动感”的关键量不是 `rhythmicity` 本身，而是：

`groove = rhythmicity * recovery_match * anticipation_success * freshness`

其中：

1. `anticipation_success`
   - 最近几拍预测是否成功
2. `freshness`
   - 防止无限重复后仍然一直爽

### 7.3 为什么这样最符合哲学

它没有硬编码：

1. 四拍
2. 八拍
3. 某种固定 BPM

而是让“偏爱的节奏”从主体自身恢复曲线中涌现。

这比写死“四拍最好听”更拟人。

## 8. 节奏期待与命中

当系统已经形成主周期 `tau` 后，定义下一次期待时刻：

`t_next = t_last + tau`

当前 tick 对该事件族的相位误差为：

`phase_error = abs(t_now - t_next)`

则节奏期待强度可定义为：

`phase_expectation = regularity * exp(-(phase_error^2)/(2*sigma_phase^2))`

这会生成：

1. `rhythmfelt::phase_expectation`
2. 并通过已有 `expectation` 通道产生期待

当新命中在 `t_next` 附近出现时：

1. 提高 `grasp`
2. 提高 `correctness`
3. 给少量奖励

当新命中缺席时：

1. 若原期待很高，则产生 `surprise`
2. 若已形成较强承诺，还应产生一定 `dissonance`
3. 并伴随少量惩罚

## 9. 节奏疲劳与“腻”

为了防止进入无限循环，节奏必须有“腻”的机制。

建议引入：

1. `rhythm_feeling_fatigue`
2. `rhythm_phase_fatigue`

逻辑如下：

1. 短期内重复命中会增强节奏感
2. 但连续完美命中太久，会累积节奏疲劳
3. 节奏疲劳会逐渐压低额外奖励和注意力优先级
4. 一旦节奏被打断，或进入空档期，疲劳快速恢复

这样就会出现更像人的图景：

1. 刚进入规律时会觉得有节奏、很顺
2. 连续太久会觉得腻
3. 停一下再回来，又会重新觉得顺

## 11. 与现有系统最自然的对接方式

建议实现顺序：

1. 基于当前 `timefelt` 体系增加 `RhythmTracker`
2. 它不直接扫全状态池，而只跟踪：
   - 高注意力对象
   - 高 commitment 对象
   - 高现实支持对象
3. 用最近窗口中的重复命中时刻构造 `tau`
4. 生成 `rhythmfelt::pulse`
5. 当逼近下一拍时生成 `rhythmfelt::phase_expectation`
6. 命中/落空继续走现有情绪与认知感受链

这样工程最稳，也最统一。

## 12. 推荐参数方向

第一版建议：

1. `rhythm_window = 8 ~ 16 ticks`
2. `rhythm_min_hits = 3`
3. `rhythm_min_period = 2 ticks`
4. `rhythm_max_period = 12 ticks`
5. `rhythm_period_sigma = 0.18 * tau`
6. `rhythm_phase_sigma = 0.22 * tau`
7. `rhythm_pulse_threshold = 0.18`
8. `rhythm_phase_threshold = 0.14`
9. `rhythm_fatigue_decay = 0.82`
10. `rhythm_fatigue_step = 0.12`
11. `rhythm_fatigue_gain = 0.55`
12. `rhythm_fatigue_max = 1.0`

如果 tick 约为 80~120ms，那么 `tau = 4~8 ticks` 会自然落在较接近人类常见节拍感区间。

但这不是硬编码偏好，只是初始工程近似。

## 13. 最终推荐结论

我推荐的最终方案是：

> 把节奏感定义为“重复事件的短期相位锁定结构”，再用疲劳恢复匹配来决定它是否真正变成拟人的节拍感和律动感。

这套方案的优点：

1. 和时间感受哲学完全同路
2. 不需要硬编码四拍、八拍
3. 能自然解释为什么短期重复比长期孤立重复更像节奏
4. 能自然解释为什么太久会腻
5. 能把命中、落空、期待、把握、惊、违和统一进现有链路

因此，这是我认为目前最优雅、最符合你哲学、也最像人的方案。
