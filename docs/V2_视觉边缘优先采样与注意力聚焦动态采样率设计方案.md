# V2 视觉边缘优先采样与注意力聚焦动态采样率设计方案

## 1. 设计目标

这轮设计解决两个问题：

1. 让图像中高对比度、边界、笔画感更强的位置，更容易被采样进入状态池。
2. 让“注意力聚焦/继续看/检查残差”这类内部行动，不只是移动视焦点，而且能在下一 tick 真正提高局部视觉采样率，形成类似人类“仔细看”的效果。

这不是传统 OCR 的工程外挂，也不是单纯把预算调大，而是继续沿用 AP V2 的统一哲学：

- 感知先进入状态池
- 行动器在当前 tick 决定“看哪里、看多仔细”
- 下一 tick 感受器按这个内在行动结果改变采样分布
- 通过连续多个 tick 的观察，逐步形成更密、更稳的视觉现状

## 2. 为什么要这样做

此前视觉采样虽然已经支持：

- 原图分辨率稀疏采样
- 视焦点附近偏置
- 焦点特征写入记忆
- 多 tick 叠加重建

但还存在两个明显缺口：

1. 边界/高对比区域虽然会在排序阶段被打高分，但它们并不真正主导候选点生成。
2. “continue_focus / move_gaze / inspect_residual” 之前只改变 gaze_center，没有把“仔细看”的力度传回视觉采样层。

结果就是：

- 关键边界有时根本没被采到
- 行动层和感受器层之间缺一条闭环

这轮的目标，就是把这两条补齐。

## 3. 新版机制总览

### 3.1 双层边缘优先

边缘优先现在分成两层：

1. 候选点分布层  
   在 `_sample_candidate_positions(...)` 中新增 edge-biased proposal。

2. 候选点排序层  
   在 `_build_sample(...)` 中提高 `edge_priority` 与 `stroke_priority` 对 `raw_priority`、`focus_priority` 的影响。

这样做的意义是：

- 不只是“采到以后更容易排前面”
- 而是“更有机会先被采到”

### 3.2 注意力聚焦动态采样率

新增 `VisionSensorV1._attention_boost` 内部状态。

当前 tick：

- `ActionPlannerV2` 选出动作
- `RuntimeV2.apply_selected_actions(...)` 执行 `move_gaze / continue_focus / inspect_residual`
- 根据动作的 `firmness_norm` 写入 `vision_sensor.apply_attention_boost(...)`

下一 tick：

- 视觉感受器读取 boost
- 提高 `raw_budget`
- 提高 `focus_priority_budget`
- 缩小 gaze 周围的有效采样半径
- 提高 edge gain
- 让 gaze 周围的采样分布更尖锐

随后：

- boost 按衰减系数逐 tick 衰减
- 若没有继续聚焦，系统自然回到基线采样状态

## 4. 时序逻辑

### 4.1 为什么是“下一 tick 生效”

这是最符合理论的顺序：

1. 当前 tick 先感知现状
2. 当前 tick 内部行动决定“我要仔细看这里”
3. 下一 tick 感受器据此调整采样

如果让当前 tick 直接因为刚决定聚焦就瞬间提高采样率，会变成一种不自然的“同 tick 魔法加料”。

所以这里故意保持：

- 当前 tick 决策
- 下一 tick 感知增强

这和具身连续系统的时间因果更一致。

## 5. 数学直觉

### 5.1 边缘优先

局部边缘分数由两部分组成：

- `local_contrast`
- `gradient_mag`

它们共同构成 edge probe score，用于候选点分布层。

之后在样本打分层，再构造：

- `edge_priority`
- `stroke_priority`

它们参与：

- `raw_priority`
- `focus_priority`

于是：

- 候选点更容易落在边界上
- 排序时边界和笔画感也更容易胜出

### 5.2 注意力 boost

由 `firmness_norm` 决定：

- `raw_budget_bonus`
- `focus_budget_bonus`
- `radius_scale`
- `edge_gain`
- `gaze_sigma_scale`

直觉上：

- 越坚决，采样越密
- 越坚决，越偏向 gaze 附近
- 越坚决，边缘/笔画偏好越强

## 6. 工程落地项

本轮已经落地：

- `VisionSensorV1`
  - 新增 edge-biased candidate proposal
  - 新增 attention boost state
  - 新增 effective sampling profile
  - 新增 export/import 持久化

- `RuntimeV2`
  - 将视觉相关 planner action 写回传感器 boost
  - 支持 `firmness_norm` 缺省回退

- `ComputerExecutorV2`
  - 保留 `firmness_norm` 到执行结果，避免执行链条丢失聚焦力度

- 配置层
  - 新增一组视觉动态采样相关参数

## 7. 当前配置项

当前默认新增参数：

- `vision_edge_candidate_gain`
- `vision_edge_priority_gain`
- `vision_attention_boost_enabled`
- `vision_attention_boost_decay`
- `vision_attention_boost_max_extra_raw_budget`
- `vision_attention_boost_max_extra_focus_budget`
- `vision_attention_boost_min_radius_scale`
- `vision_attention_boost_edge_gain`
- `vision_attention_boost_gaze_sigma_scale`

这些参数都保持“调制”风格，而不是硬编码某个具体视觉行为。

## 9. 与你的哲学的一致性

这版设计和你之前说的方向是一致的：

1. 不是传统 OCR
2. 不是固定分辨率感知层先做重处理
3. 不是外部规则直接替代 AP
4. 而是：
   - 原图分辨率采样
   - 状态池承接大量现状
   - 少量焦点特征写记忆
   - 行动器决定“是否仔细看”
   - 感受器下一 tick 改变采样密度

这就把“惊鸿一瞥”和“盯着细看”的差别真正做进了系统。

## 10. 下一步建议

这轮做完后，下一步最值得继续做的是：

1. 复跑 OCR-like 探针，观察边缘优先采样后，识别阈值是否下降
2. 前端把 gaze / raw sample / focus sample / edge-rich region 可视化
3. 将同样的范式推广到：
   - 听觉的“仔细听”
   - 文本的“仔细想”

也就是说，这轮不是孤立修视觉，而是在给更一般的“注意力 -> 感受器采样增强”机制打底。
