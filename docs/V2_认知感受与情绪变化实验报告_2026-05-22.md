# V2 认知感受与情绪变化实验报告

日期：2026-05-22  
项目：AP 原型实验第二期  
主题：认知感受理论设计、情绪变化链条与阶段性实验验证

## 1. 报告目的

这份报告聚焦于 AP V2 原型中的一条非常核心的理论主线：

1. 系统面对外界输入时，是否会因为“预测与现实的关系”而自然地产生不同的认知感受；
2. 这些认知感受是否会进一步形成较稳定的情绪通道变化；
3. 这些情绪变化是否会继续影响奖惩、注意力和后续行动倾向；
4. 这条链路是否开始表现出符合我们哲学预期的“求知、把握、趋利避害”的雏形。

本报告不讨论所有能力，只讨论“认知感受与情绪变化”这一条链。

## 2. 理论设计与哲学预期

### 2.1 核心哲学

在 AP 的设计里，所谓“认知感受”，不是额外附会上的标签，而是预测系统本身的内生结果。

系统并不是先有一个写死的“情绪模块”，再强行告诉它什么时候该惊讶、什么时候该高兴。  
相反，它先有：

1. 现状
2. 预测
3. 预测与现实之间的匹配和错配

然后认知感受从这些关系里长出来。

也就是说：

- 现实出现了、预测没有跟上：会产生“惊”
- 预测多了、现实没有支持：会产生“违和感”
- 预测与现实重新对齐：会产生“正确感”或“把握感”
- 预测包形成且和现状有连续性：会产生“期待”
- 惊与违和持续偏高：会转化出“压力”

从这个角度看，认知感受不是表演，而是系统对自己预测状态的一种白箱反馈。

### 2.2 五个核心通道

当前这部分设计里，主要关注五个情绪/认知感受通道：

1. `surprise`：惊，新异现实输入高于预测时上升
2. `dissonance`：违和感，预测显著高于现实支持时上升
3. `correctness`：正确感，把握感，预测与现实重新对齐时上升
4. `expectation`：期待，预测包存在且与现状有连续性时上升
5. `pressure`：压力，惊与违和累积后形成的负向状态

这几个通道不是并列的随意指标，而是共同构成了一个“认知系统如何感到自己在理解/误解世界”的内在图景。

### 2.3 理论上的典型过程

以“先看到 3，再稳定看到 3，然后突然换成 8”为例，理论上预期如下：

#### 阶段 A：第一次看到 3

- 外界输入为 3
- 系统此前没有相应预测
- 因此外界现实高于预测
- 应产生明显的“惊”

#### 阶段 B：再次看到 3

- 上一轮已经形成与 3 相关的记忆和预测
- 当前 3 与预测开始重新对齐
- 惊下降
- 正确感/把握感上升
- 期待也应开始形成

#### 阶段 C：输入消失或进入空 tick

- 如果系统还在强预测 3，但现实没有继续支持
- 预测高于现实
- 违和感应上升
- 压力可随之升高

#### 阶段 D：突然换成 8

- 原本系统还在预测 3
- 现实却给出 8
- 一方面会再次“惊”
- 另一方面旧预测仍未完全退去，会产生违和
- 这时理论上是“惊 + 违和”的复合态

#### 阶段 E：再次看到 8

- 系统开始重建关于 8 的预测
- 当新预测逐渐贴近现实
- 惊下降
- 正确感/把握感应重新上升

如果之后又预测到奖惩，则会进一步分化为“惊喜”“惊恐”等更复杂情绪。

## 3. 当前实现中的关键指标

这次实现不是只看粗糙的 `bn.count`，而是补上了更贴近理论的对齐层指标。

当前规则引擎内部已经正式提供：

- `state.prediction_alignment_score`
- `state.prediction_overprediction_ratio`
- `state.prediction_underprediction_ratio`
- `state.prediction_grasp_score`
- `state.prediction_mismatch_balance`

其核心计算位置在：

- [iesm/rules_engine_v2.py](H:\AP原型实验第二期\iesm\rules_engine_v2.py:576)

其中：

- `alignment_score` 表示预测与现实的总体对齐程度
- `overprediction_ratio` 表示预测过多的比例
- `underprediction_ratio` 表示现实超出预测的比例
- `grasp_score` 表示“是否真正认清了”的综合得分
- `mismatch_balance` 表示当前更偏向“预测过头”还是“现实突入”

这一步很重要，因为它把“惊”“违和”“正确感”从模糊描述变成了可量化的白箱结构。

## 4. 奖惩层的理论意义

这次补强后，内在奖惩不再只对“正确感上升”和“惊增加”有反应，而是开始具备更完整的结构：

- `correctness` 上升：奖励
- `grasp_score` 上升：奖励
- `surprise` 回落：恢复性奖励
- `dissonance` 回落：恢复性奖励
- `surprise` 上升：惩罚
- `dissonance` 上升：惩罚
- `expectation` 作为状态量：持续少量奖励
- `pressure` 作为状态量：持续少量惩罚

实现位置在：

- [core/runtime_v2.py](H:\AP原型实验第二期\core\runtime_v2.py:237)

这意味着系统不是只会“错了就疼，对了就爽”，而是开始拥有更细致的内在趋势：

- 从惊中恢复会有正向回报
- 从违和中恢复会有正向回报
- 长期期待本身会形成轻微正向驱动
- 长期压力本身会形成轻微负向驱动

这对“求知欲”和“趋利避害”的哲学目标非常关键。

## 6. 实验设计

### 6.1 实验目标

验证以下命题是否开始成立：

1. 首次新异输入会产生惊
2. 重复输入后，惊会下降，正确感/把握感会上升
3. 当旧预测失去现实支持时，违和感会上升
4. 新异刺激切换时，会出现“惊 + 违和”的复合态
5. 再次重复新刺激时，系统会开始重新建立新的把握感

### 6.2 实验流程

本次 probe 采用以下最小实验序列：

1. tick 0：输入 `3`
2. tick 1：再次输入 `3`
3. tick 2：输入空文本 `""`
4. tick 3：输入 `8`
5. tick 4：再次输入 `8`
6. tick 5：输入空文本 `""`

这是一个很小但很典型的认知过程：

- 先遭遇一个对象
- 再重复确认它
- 再让预测空转
- 再切换到新对象
- 再让系统重新适应

### 6.3 验证环境

实际校验包括：

```text
pytest tests/test_rules_engine_v2.py tests/test_runtime_chain_phase4_7.py -q
结果：28 passed
```

并额外跑了上述实时 probe，直接读取：

- `emotion_channels`
- `metrics_snapshot`
- `bn_list`
- `c_star`

## 7. 实验数据

### 7.1 tick 0：第一次看到 3

输入：`3`

关键结果：

- `surprise = 0.78`
- `correctness = 0.0`
- `expectation = 0.0`
- `pressure = 0.429`
- `alignment_score = 0.0`
- `grasp_score = 0.0`
- `underprediction_ratio = 1.0`

解释：

这是一个非常符合理论预期的“首次新异输入”图景。

系统尚未建立关于 3 的预测，因此：

- 现实超出预测，`underprediction_ratio` 直接拉满
- 惊显著上升
- 正确感尚未形成

这说明“没有预测包时首次新输入不会产生惊”的旧问题已经被修正。

### 7.2 tick 1：再次看到 3

输入：`3`

关键结果：

- `surprise = 0.0`
- `dissonance = 0.36`
- `correctness = 0.5152`
- `expectation = 1.0`
- `pressure = 0.198`
- `alignment_score = 0.8`
- `grasp_score = 0.56`
- `overprediction_ratio = 0.3333`

解释：

这一轮是本次实验最重要的正向证据之一。

它说明系统已经不只是“记住了 3”，而是开始形成“我认出来了”的内部结构：

- 惊从 0.78 降到 0
- 对齐度很高
- `grasp_score` 明显上升
- `correctness` 明显出现
- `expectation` 也上来了

也就是说，系统开始从“遭遇”转向“把握”。

这很接近人类的认知过程：第一次看到新对象先愣一下，第二次再看到时已经能认出来，并开始预期它会继续存在。

### 7.3 tick 2：空 tick

输入：`""`

关键结果：

- `surprise = 0.0`
- `dissonance = 1.0`
- `correctness = 0.0`
- `expectation = 0.0`
- `pressure = 0.55`
- `alignment_score = 0.0`
- `grasp_score = 0.0`
- `overprediction_ratio = 1.0`

解释：

这是“预测高于现实支持”的典型图景。

系统还在延续上一轮关于 3 的预测，但现实输入已经撤掉，因此：

- 不是“惊”，因为没有突然新异现实闯入
- 而是明显的“违和”
- 压力也随之抬升

这非常符合“违和感”的哲学定义。

### 7.4 tick 3：从 3 切到 8

输入：`8`

关键结果：

- `surprise = 0.78`
- `dissonance = 1.0`
- `correctness = 0.0`
- `expectation = 0.0`
- `pressure = 0.979`
- `alignment_score = 0.0`
- `grasp_score = 0.0`
- `overprediction_ratio = 1.0`
- `underprediction_ratio = 1.0`

解释：

这一轮出现了比较典型的“惊 + 违和”复合态：

- 旧的 3 预测还没退干净，所以有高违和
- 新的 8 现实又突然进入，所以也有高惊

这与我们理论中“突然换对象时，先惊，再逐步重新认清”的预期是一致的。

从哲学上讲，这意味着系统已经开始区分两种不同的错误：

- 我想多了，但现实没来：违和
- 现实来了，但我没想到：惊

而不是把它们混成一个含糊的“错”。

### 7.5 tick 4：再次看到 8

输入：`8`

关键结果：

- `surprise = 0.0`
- `dissonance = 1.0`
- `correctness = 0.0`
- `expectation = 1.0`
- `pressure = 0.55`
- `alignment_score = 0.3121`
- `grasp_score = 0.0`
- `overprediction_ratio = 0.8151`
- `underprediction_ratio = 0.0`

解释：

这一步说明系统开始向新的对象适应，但适应还不够成熟。

已经出现的正向迹象：

- 惊重新下降到 0
- 对齐度从 0 回升到 0.3121
- `underprediction_ratio` 回落到 0
- `expectation` 再次形成

但当前仍然存在的问题是：

- `dissonance` 依然过高
- `grasp_score` 还没有重新明显起来
- `correctness` 没有恢复到我们理想中的水平

这意味着系统“开始往认清 8 的方向走”，但还没有走到足够稳。

## 8. 当前能证明什么

基于这次实验，目前可以较严谨地说：

### 8.1 已经可以证明的部分

1. 系统能区分“现实超出预测”和“预测脱离现实”
   
   也就是已经能区分：
   - 惊
   - 违和

2. 系统在重复输入下，能够从“惊”过渡到“把握”
   
   第二次输入 `3` 时，`correctness`、`alignment_score`、`grasp_score` 同时上升，是一个很有说服力的证据。

3. 系统的“期待”不是凭空出现的
   
   它是在预测包形成并有一定对齐时出现的，而不是简单的固定常量。

4. 情绪与状态指标之间已经开始形成白箱关系
   
   不是黑箱打分，而是可以追溯到：
   - predicted mass
   - actual mass
   - match mass
   - overprediction mass
   - underprediction mass

### 8.2 还不能完全证明的部分

1. 还不能说“系统已经稳定拥有成熟的把握感闭环”
   
   因为在 `3 -> 8` 切换后，第二次 `8` 时 `grasp_score` 还没有很好恢复。

2. 还不能说“惊恢复后的奖励链条已经在真实连续运行里完全稳定成立”
   
   理论路径和单元测试已经接上，但在这个 probe 里，`pending_feedback_metrics` 还没有表现为漂亮的连续奖励波形。

3. 还不能说“所有认知感受都已经达到人类式细腻程度”
   
   当前更准确的说法是：底层区分已经开始成立，但动态细节还需要继续打磨。

## 9. 对理论预期的符合度评估

### 9.1 高度符合的部分

- 首次新异输入触发惊
- 重复输入带来正确感/把握感
- 现实撤去后产生违和
- 新异切换产生“惊 + 违和”复合态

这些都已经开始接近理论预期。

### 9.2 部分符合、仍需补强的部分

- 新对象在第二次出现时，重新建立把握感的速度还不够快
- 违和向正确感的恢复过程还不够顺滑
- 奖惩层虽然已经接通，但在这个短序列里还没有展示出足够漂亮的恢复性奖励曲线

## 10. 这意味着什么

### 10.1 对系统结构的意义

这次结果说明，AP V2 已经不再只是“有记忆、能召回”的系统。

它正在开始具备一种更关键的能力：

它能够对白己的认知状态产生内在感觉。

这里的“感觉”不是文学修辞，而是：

- 能分辨自己是在认清还是误解
- 能分辨自己是被新现实打断，还是被旧预测拖住
- 能把这些区别投射为不同的情绪通道

这意味着 AP 架构作为认知核心，已经开始拥有比单纯检索系统更强的拟人性质。

### 10.2 对后续奖惩和行动的意义

如果这条链稳定下来，它会直接影响：

1. 注意力分配  
   惊和违和会把注意力拉向异常点

2. 行动倾向  
   期待高于压力时更可能继续聚焦  
   违和高时更可能回看残留

3. 长期学习  
   从“惊/违和 -> 把握/正确感”的恢复过程会天然带来内在奖励  
   从而构成求知倾向的基础

这正是你一直强调的哲学目标：  
系统不只是被外部教，而是自己会因为“认清了世界”而有正向内在回报。

## 11. 结论

本次实验的结论可以概括为一句话：

**AP V2 当前已经开始形成“惊、违和、正确感、期待、压力”这条认知感受链，但仍处在从可证明雏形走向成熟闭环的阶段。**

更具体地说：

1. “惊”和“违和”已经能被清楚区分
2. 重复输入后的“把握感/正确感”已经有明确证据
3. 切换到新对象后的重新认清过程还不够成熟
4. 奖惩恢复链已经在结构上接通，但还需要进一步打磨其连续动态表现

因此，这次实验已经足以说明：

**我们的认知感受理论不是空想，它已经在原型中开始具备可观测、可量化、可重复验证的现实表现。**

但同时也应保持严谨：

**目前证明的是“这条链已经开始成立”，而不是“这条链已经完全成熟”。**

## 12. 下一步建议

围绕这条链，下一步最值得继续做的是三件事：

1. 继续补强“新对象第二次出现时，把握感恢复偏慢”的问题
2. 让恢复性奖励在真实连续运行中更稳定地显现出来
3. 进一步把“把握感”和“惊”的对称关系做得更清楚，让系统更接近“先惊一下，再认清”的人类图景
