# AP 二期 V2 阶段性实验评估报告

日期：2026-05-20

## 1. 摘要

这份报告基于当前仓内已经完成的 V2 实测结果，对以下六类能力进行阶段性评估：

1. 组织比较有逻辑性的语言及相关示例
2. 同义词学习
3. 文本抽象能力
4. 多模态统一理解与混合召回能力
5. 行动奖惩的海豚训练
6. 认知感受的产生预期

本轮结论可以先直接说在前面：

1. 纯文本环境下，V2 已经出现“连续可读、局部有逻辑”的短跨度语言组织雏形，这不是主观印象，而是有结构化实验数据支持的。
2. 预测-实际错配驱动的“同义近义桥接”已经出现符合预期的证据，说明系统并不是只能死记原句，而是会在预测失败中把相近表达拉近。
3. 基于共同部分的实时抽象信号已经出现，说明它虽然主要保存记忆，但并非完全缺乏在线抽象能力。
4. 多模态绑定与跨模态召回雏形已经被证明：文本、图像采样、音频采样可以在同一时空邻域中形成绑定，随后由任一模态线索唤起其它模态相关记忆。
5. 行动奖惩链条已经不只是“有接口”，而是已经在结构化探针中形成可量化的条件动作学习结果，海豚训练实验当前通过。
6. 奖励/惩罚与期待/压力、正确感/违和感之间的链条已经出现稳定实测支持，说明“求知欲”“趋利避害”这套哲学方向至少在原型层面已经有了可复现实验依据。

但同样需要严谨强调：

1. 这些结果证明的是“雏形能力、原理可行性、架构潜力”，不是终局产品态。
2. 当前还不能声称它已经达到完整的人类语言能力、成熟多模态语义理解、真实世界长期具身自主智能。
3. 当前最有价值的意义在于：V2 的底层链条已经足够闭合，关键哲学目标中最核心的几条，已经开始从理论草案进入可复现证据阶段。

## 3. 理论意义总览

如果把你最关心的理论目标转成更明确的话，那么这轮实验的意义可以这样理解：

1. 如果“语言组织雏形 + 同义桥接 + 文本抽象”成立，那么 V2 就不只是一个记忆回放器，而是已经显示出和 transformer 路线在语言组织问题上进行比较与竞争的资格。
2. 如果“多模态统一绑定与召回”成立，那么它就更接近“教一个人类认识苹果”的方式：同时给文字、图像、声音，之后看到其一便能联想到其余。
3. 如果“奖惩-情绪-行动塑形”成立，那么系统就不只是被动回忆，而是具备了方向性的自适应基础，能够在确定环境中逐步改善自己的处境。
4. 如果“违和感/正确感/期待/压力”能够与预测和反馈稳定耦合，那么求知欲与趋利避害就不再只是口号，而是有望成为系统长期发展中的真实驱动力。

从这个角度看，V2 当前最重要的阶段性成果不是“已经完美拟人”，而是“拟人化心智核心所需要的若干关键链条，开始被独立地、可复现地证明为可运行”。

## 4. 实验一：组织比较有逻辑性的语言

## 4.1 实验目标

验证在纯文本连续输入、且夹杂空 tick 的环境中，系统是否能够依靠短期记忆、长期记忆、后继优势与预测竞争，逐步组织出相对连续、可读、局部有逻辑的语言链条。

这项实验的目标不是检验它是否已经成为成熟语法模型，而是看它是否已经拥有“像人在想事情时那样，一轮一轮 tick 延续短句片段”的雏形。

## 4.2 数据集设计

主实验脚本中，文本大型数据集包含大量带重复又不完全相同的句子片段，例如：

1. `冬天 的 天气 有点 冷`
2. `我 觉得 今天 有点 冷`
3. `冬天 的 天气 有点 凉`
4. `今天 的 温度 有点 凉`
5. `这种 感觉 有点 凉`
6. `如果 下雨 ...`
7. `如果 刮风 ...`

设计上同时保留：

1. 高重复结构
2. 局部替换词
3. 语义相邻但不完全一致的表达
4. 空 tick 无输入阶段

这样更接近真实环境里的连续认知流，而不是一次性整句分类任务。

## 4.3 关键结果

文本实验核心数据如下：

1. `tick_count = 170`
2. `focus_chain_readable_count = 170`
3. `passed_language_chain = true`
4. `mean_adjacent_lexical_overlap = 2.8284`

这意味着在 170 个 tick 中，系统每个 tick 都出现了可提取的、连续可读的焦点链片段。

## 4.4 典型示例

`focus_chain_examples` 中出现的典型片段包括：

1. `冬天的气`
2. `天气的冷`
3. `我觉得今`
4. `天冷有点`
5. `今天的温`
6. `这种感觉`
7. `今天气不`
8. `如果下雨`
9. `如果刮风`

这些片段显然还不是完整、流利、长跨度的自然语言生成结果，但它们已经不是纯随机词元堆积。它们表现出两个重要特征：

1. 片段内部大多与训练语境一致，具有局部顺序正确性。
2. 相邻 tick 之间保持一定延续性，而不是每轮都从零散乱跳。

因此，更准确的表述应该是：

当前 V2 已经具备“短跨度连续可读、局部有逻辑的语言组织雏形”，而不是“已经具备完整自然语言生成能力”。

## 4.5 结论

这一项可以判定为：**已通过阶段性证明**。

证明的不是它已经能替代成熟大语言模型，而是：

1. 在没有 transformer 自回归解码器的前提下，
2. 仅依赖状态池、注意力竞争、记忆召回、后继优势和短期连续性偏置，
3. 系统已经可以形成短程的连续语言组织能力。

## 4.6 对应意义

这件事的意义非常大。因为它说明：

1. 语言连续性并不一定只能靠标准 token-by-token transformer 解码实现。
2. 基于记忆、预测、竞争、显著性和时间邻接，也可以自然地产生相对连续的语言流。
3. 这让 AP 架构具备了与现有主流语言架构进行理论比较的资格。

## 4.7 当前边界

当前仍不能证明：

1. 长篇稳定生成
2. 严格语法控制
3. 强逻辑推理文本生成
4. 复杂多句规划

因此，当前最好把这项成果定位为：

“语言组织能力的雏形已经成立，且具有独立架构意义。”

## 5. 实验二：同义词学习

## 5.1 实验目标

验证系统是否能够通过“预测-实际错配”机制，把语义相近但表达不同的词项在记忆中拉近，从而逐渐形成同义/近义桥接。

这比单纯词向量相似更重要，因为这里关注的是：

1. 先有预测
2. 再有现实输入与预测不一致
3. 错配引起违和感和注意增强
4. 预测项与实际项在近时空共同进入记忆
5. 以后再提到其中一个，就更容易联想到另一个

这更符合你之前设想的人类式学习方式。

## 5.2 数据集设计

文本实验中反复混用以下表达：

1. `冬天 的 天气 有点 冷`
2. `冬天 的 天气 有点 凉`
3. `今天 的 温度 有点 凉`
4. `这种 感觉 有点 凉`

也就是说，系统常常先学到“有点冷”的预测，然后现实输入变成“有点凉”。

## 5.3 关键结果

关键指标包括：

1. `passed_synonym_mismatch_signal = true`
2. `synonym_bridge_hits = 11`
3. `mismatch_tick_count = 168`

其中 `synonym_bridge_hits = 11` 说明在明确设计的近义错配场景里，已经出现了预期中的桥接命中。

## 5.4 典型示例

例如在 `tick_index = 6` 的例子中：

输入为：

`今天 的 温度 有点 凉`

而预测侧仍出现：

1. `冷`
2. `有点`
3. `天`
4. `气`
5. `的`

实际侧则出现：

1. `今天`
2. `温`
3. `度`
4. `有点凉`
5. `凉`

这正是你理论里最关键的一幕：

1. 系统根据历史习惯预期“冷”
2. 现实却给出“凉”
3. 两者在同一轮错配中同时被加强
4. 从而使“冷”与“凉”在后续联想中更容易互相靠近

## 5.5 结论

这一项可以判定为：**已出现符合理论预期的阶段性证明**。

准确地说，当前证明的是：

1. 系统已经出现“预测项 - 实际项”桥接证据
2. 这种桥接可以把近义表达通过错配联系起来
3. 它不是只会记住完全一致的表面字符串

## 5.6 对应意义

这项结果的意义，可以直接用你最关心的类比来说明：

如果我们教它“这是苹果”，同时让“苹果”这个文字、苹果的图像、苹果相关声音共同出现，那么未来它看到苹果图像、听到相关声音、或者见到“苹果”这个文字时，就都有机会把其它模态的信息一起联想到。

在文本内部，“冷”和“凉”的桥接是这个更一般能力的一个缩影：它表明系统能够通过实际经验，把“本来预测的东西”和“现实真正出现的东西”建立联系。

## 5.7 当前边界

当前还不能证明：

1. 已形成成熟的同义词词典
2. 能稳定处理大规模开放词汇同义网络
3. 能做严格的词义消歧

目前最稳妥的表述是：

“同义/近义桥接机制已经出现初步可复现实证。”

## 6. 实验三：文本抽象能力

## 6.1 实验目标

验证系统虽然主要保留的是记忆信息，但是否已经能在多次相似输入中，把共同部分在线提炼出来，形成一定的抽象片段。

## 6.2 关键结果

关键指标如下：

1. `passed_abstraction_signal = true`
2. `dynamic_phrase_hit_count = 1467`
3. `dynamic_phrase_labels` 中出现了大量重复共同部分

代表性的动态短语包括：

1. `phrase::今_天`
2. `phrase::今_天_的`
3. `phrase::冬_天`
4. `phrase::冬_天_的`
5. `phrase::下_雨`
6. `phrase::下_雨_就`
7. `phrase::刮_风`
8. `phrase::刮_风_就`

## 6.3 解释

这些结果说明系统不是只把每次完整句子原封不动堆起来，而是会在多次相似经验中，把共同出现的片段变成更高频、更稳定的结构候选。

例如：

1. 多次出现“今天 ...”
2. 多次出现“冬天 的 ...”
3. 多次出现“下雨 就 ...”
4. 多次出现“刮风 就 ...”

之后系统内部就更容易形成这些共同部分的抽象短语。

## 6.4 结论

这一项可以判定为：**已通过阶段性证明**。

但这里也要避免夸大。当前证明的是：

1. 存在基于共同部分的在线抽象信号
2. 这种抽象能在记忆组织与召回中起作用
3. 它不等于已经实现成熟概念系统或高层符号推理

## 6.5 对应意义

这件事的重要性在于：如果一个系统只有记忆而没有抽象，那么它会很快沦为低效的经验堆积器。当前出现的抽象信号说明：

1. V2 不是纯粹死记硬背
2. 它可以从共同部分中生长出更稳定的结构
3. 这让它有机会在未来扩展出更强的概念化能力

## 6.6 当前边界

当前尚不能证明：

1. 抽象概念的层级递归建模
2. 高层语义范畴系统
3. 强形式逻辑概念运算

目前最稳妥的结论是：

“实时抽象雏形已经成立，但仍处于较浅层、较局部的阶段。”

## 7. 实验四：多模态统一理解与混合召回能力

## 7.1 实验目标

验证文本、图像采样、音频采样是否能够在同一记忆链中绑定，并在后续通过单一模态线索召回其它模态相关内容。

这项实验对应你非常看重的能力：

不是“图像模型”“音频模型”“文本模型”各干各的，而是同一套心智核心把它们共同放进同一片时空记忆里。

## 7.2 数据集设计

主实验脚本里构造了一个小型混合输入数据集，核心模式包括：

1. `红 苹果 叫` + 红色苹果图像 + 铃声样音频
2. `黄 香蕉 响` + 黄色香蕉图像 + 低音样音频
3. 单独输入 `红 苹果`
4. 单独输入 `叫`
5. 单独输入 `黄 香蕉`
6. 单独输入 `响`

这类设计的目的是：

先把多模态对象共同绑定，再看其中任一模态是否能在后续召回另两个模态相关记忆。

## 7.3 关键结果

多模态实验核心数据如下：

1. `tick_count = 30`
2. `multi_modal_memory_ticks = 30`
3. `cross_modal_candidate_ticks = 68`
4. `true_cross_modal_hits = 21`
5. `passed_cross_modal_binding = true`

## 7.4 典型示例

### 示例 A：输入 `红 苹果`

在 `tick_index = 2` 时，输入是：

`红 苹果`

其 `bn_preview` 召回的候选中，前列出现：

1. `红 苹果 叫`
2. `红苹果叫`

这说明仅凭文本加视觉线索，就能回忆起曾经一起绑定过的“叫”这一声音/文本相关记忆。

### 示例 B：输入 `叫`

在 `tick_index = 3` 时，输入是：

`叫`

其 `bn_preview` 同样召回：

1. `红 苹果 叫`
2. `红苹果叫`

这说明单独的音频/文本线索，也会把和该声音共同出现过的对象整体回忆出来。

### 示例 C：输入 `黄 香蕉`

在 `tick_index = 4` 时，输入是：

`黄 香蕉`

其召回前列出现：

1. `黄 香蕉 响`
2. `黄香蕉响`

这说明第二组模态绑定也独立成立。

## 7.5 结论

这一项可以判定为：**已通过阶段性证明**。

当前已经可以严谨地说：

1. 文本、图像采样、音频采样可以被绑定在同一记忆结构中
2. 单一模态线索可以跨模态触发相关记忆召回
3. 这不是只看起来像，而是已经在结构化实验中出现了可量化命中

## 7.6 对应意义

这项实验是 V2 当前最令人兴奋的结果之一。因为它说明：

1. 多模态统一记忆并非只能靠外部大模型做后融合
2. AP 这套架构本身已经能把多模态经验共同写入同一认知主链
3. 这非常接近具身智能所需的“统一心智核心”

如果要向同事更直观地解释，可以这么说：

“我们不是分别训练一个文本脑、一个视觉脑、一个听觉脑，而是在同一套状态池-记忆-预测-奖惩结构里，让它像人一样，把同一件事的多种感受绑在一起。”

## 7.7 当前边界

当前仍不能证明：

1. 已具备成熟多模态语义理解
2. 能重建高保真图像或音频
3. 能构成完整三维时空世界模型
4. 连续视频流、连续麦克风流、双耳定位已经完全成立

所以当前最准确的结论是：

“跨模态绑定与跨模态召回雏形已经成立，但仍处于原型阶段。”

## 8. 实验五：行动奖惩的海豚训练

## 8.1 实验目标

验证系统是否能在明确奖惩反馈环境中，不只是被动记录，而是主动塑形自己的行动偏好，使“正确动作”越来越稳定，“错误动作”越来越不容易被选中。

这就是你说的“海豚训练”。

如果这条成立，说明系统已经具备一种非常关键的能力：

它不是只会想，还会在环境反馈中学着做。

## 8.2 两层证据结构

这一项我们现在有两层证据：

1. 主实验套件里的自主回路学习结果
2. 更严格的条件动作学习探针结果

前者证明“整体学习链条已通”，后者证明“带参数动作在奖惩环境中已可被明确塑形”。

## 8.3 主实验套件结果

主套件中两组典型任务分别是：

1. `海豚训练_记事本`
2. `海豚训练_计算器`

其结果为：

1. `海豚训练_记事本`
   - `expected_action = type_text`
   - `dominant_action = type_text`
   - `selected_action_count = 18`
   - `feedback_reward_total = 5.44`
   - `passed = true`
2. `海豚训练_计算器`
   - `expected_action = press_key`
   - `dominant_action = press_key`
   - `selected_action_count = 18`
   - `feedback_reward_total = 4.08`
   - `passed = true`

这说明在自主循环环境中，系统已经能在反复奖惩后，把目标动作变成主导动作。

## 8.4 更严格的条件动作学习探针

为了避免“只是因为默认偏置凑巧成功”，本轮又额外使用了一个更严格的探针：

1. `口令甲` 的正确动作是 `press_key:left`
2. `口令乙` 的正确动作是 `press_key:right`

而在训练前，系统对两者并没有都学会。探针总结结果如下：

1. `baseline_probe_accuracy = 0.5`
2. `trained_probe_accuracy = 1.0`
3. `alpha_probe_left_hits = 12 / 12`
4. `beta_probe_right_hits = 12 / 12`
5. `passed_conditioned_action_learning = true`

这意味着：

1. 训练前它只稳定掌握了一半映射
2. 训练后两种口令都能稳定映射到正确动作
3. 这已经不再是“感觉像学会了”，而是明确的、可量化的条件动作学习成功

## 8.5 典型示例

### 训练前

探针中 `口令乙` 的基线例子会出现：

1. `expected_signature = press_key:right`
2. `selected_signature = press_key:left`

也就是说，它还会做错。

### 训练后

训练后则出现：

1. `口令甲 -> press_key:left`
2. `口令乙 -> press_key:right`

并且两者命中率达到：

1. `口令甲：12/12`
2. `口令乙：12/12`

## 8.6 结论

这一项当前可以判定为：**已通过阶段性证明**。

而且这一次的证据强度比之前更高，因为我们已经不只是有“整体 run 通过”，还多了一个专门针对条件动作学习的对照探针，明确显示：

1. 训练前不会
2. 训练后会了
3. 正确率从 `0.5` 提升到 `1.0`

## 8.7 对应意义

这项结果的重要意义在于：

1. AP 不只是会“联想”或“回忆”
2. 它已经能在奖惩环境中塑形自己的行动选择
3. 这意味着它的发展开始具有方向性
4. 它可以通过自身行动经验来改善自己的处境

从具身智能的角度看，这一点非常关键。因为一个真正的认知核心，不应该只在脑内做静态匹配，而要能在环境反馈中改变行为。

## 8.8 当前边界

当前仍不能证明：

1. 真实开放世界里的长期复杂自主任务学习
2. 多阶段任务保持与自恢复
3. 连续数小时甚至数天的稳定自主策略形成
4. 高维复杂动作空间下的大规模泛化

但在“确定环境 + 明确奖惩信号 + 可定义正确动作”的条件下，海豚训练链条当前已经成立。

## 9. 实验六：认知感受的产生预期

## 9.1 实验目标

验证违和感、正确感、期待、压力是否真的和预测命中、预测失败、奖励、惩罚发生了符合理论预期的耦合，而不是只是名字写在草案里。

这项实验非常关键，因为它决定了“求知欲”“趋利避害”能否从哲学意图变成真实动力。

## 9.2 主实验套件结果

在主实验套件中，情绪链条给出如下结果：

1. `passed_emotion_signal = true`
2. `expectation_minus_pressure_total = 0.5155`
3. `correctness_minus_dissonance_total = 1.0964`

两组海豚训练任务的均值如下：

### 海豚训练_记事本

1. `mean_dissonance = 0.2903`
2. `mean_correctness = 0.8889`
3. `mean_expectation = 0.2844`
4. `mean_pressure = 0.0`

### 海豚训练_计算器

1. `mean_dissonance = 0.5022`
2. `mean_correctness = 1.0`
3. `mean_expectation = 0.2311`
4. `mean_pressure = 0.0`

同时规则日志中反复出现：

1. `预测与实际不一致，违和感上升`
2. `预测命中，正确感上升`
3. `奖励信号抬升期待`

## 9.3 条件动作学习探针中的补强证据

更严格的条件动作学习探针进一步给出：

1. `reward_follow_expectation_mean = 1.0`
2. `reward_follow_pressure_mean = 0.0`
3. `punishment_follow_expectation_mean = 0.0`
4. `punishment_follow_pressure_mean = 1.0`
5. `punishment_follow_dissonance_mean = 1.0`
6. `reward_follow_correctness_mean = 0.9474`
7. `passed_punishment_pressure_signal = true`
8. `passed_reward_expectation_signal = true`

这说明：

1. 奖励之后，期待明显占优
2. 惩罚之后，压力明显占优
3. 惩罚之后，违和感也明显抬升
4. 奖励之后，正确感保持高位

## 9.4 结论

这一项当前可以判定为：**已通过阶段性证明**。

更具体地说，当前已经出现稳定证据支持以下链条：

1. 预测失败 -> 违和感上升
2. 预测命中 -> 正确感上升
3. 奖励反馈 -> 期待上升
4. 惩罚反馈 -> 压力上升

## 9.5 对应意义

这项结果的意义非常深。因为它说明：

1. 系统内部已经开始出现“为什么要继续学”“为什么要避开某种行为”的动力学基础
2. 求知欲不必被硬编码成一句外部目标，而可以来自预测误差与正确命中的持续调节
3. 趋利避害也不必只是外层 if/else，而可以通过奖励/惩罚、期待/压力、行动驱动力调制逐步形成

也就是说，V2 当前已经不仅是“会记、会想、会联想”，而是开始具备“会在内部感到某些东西更对、更错、更值得追求、更值得回避”的雏形。

## 9.6 当前边界

当前仍不能证明：

1. 长期真实意义上的稳定人格化情感系统
2. 开放环境下复杂价值体系的自洽形成
3. 情绪长期调控与高阶动机之间的完全闭环

但至少在当前原型层面，认知感受已经不是空设定，而是有了与预测和反馈相连的真实运行证据。
