# V2 音频结构通道调研与新版设计建议报告

日期：2026-05-23

## 1. 目标

这份报告回答一个核心问题：

> 如果希望 AP 的音频识别效果明显提升，是否也应像视觉一样，引入“结构化、可解释、连续值”的多种音频特征通道？

结论先说：

**是，而且非常有必要。**

但这里说的“加音频特征通道”，不是把 AP 降级成传统黑箱音频分类器，而是：

1. 让听觉也像视觉一样，具有若干先天偏置的、可解释的结构通道；
2. 这些通道输出连续信号，而不是硬标签；
3. 当这些通道的信号足够强时，它们也可以像时间感、违和感、惊、节奏感那样，进入状态池成为一等公民 SA；
4. 它们随后还能反过来调制召回、注意力和行动。

这与当前二期已经冷保存的统一通道哲学完全一致，而不是偏离它。

---

## 2. 当前 hearing_sensor_v1 已经做到了什么

先看现状。当前仓库里的听觉感受器并不是空白底座，而是已经具备一层相当像样的“基础听觉”。

现有实现位置：

- [sensors/hearing_sensor_v1.py](H:/AP原型实验第二期/sensors/hearing_sensor_v1.py:26)

当前已经落地的能力，按意义可以分为五类：

### 2.1 基础能量与频谱粗结构

- `rms`
- `zero_cross_rate`
- `low_band / mid_band / high_band`
- `dominant_hz`
- `spectral_centroid_hz`
- `dominant_band_index`

这些相当于视觉里“亮度、粗颜色分布、主要方向”的第一层底座。

### 2.2 音频焦点

- 可移动的 `audio_focus`
- 频带范围控制 `focus_bandwidth_octaves`
- 焦点邻近增强 `focus_bonus`
- 注意力驱动的听觉聚焦 `apply_attention_boost(...)`

这已经很接近你想要的“听觉焦点”设计了：不是在二维平面移动，而是在频率轴附近提高采样权重。

### 2.3 变化与新异性

- `delta`
- `spectral_similarity`
- `onset_strength`
- `novelty`

这部分已经在做“和近期缓存相比，有什么变化”，相当于动态感知的第一层。

### 2.4 去重、疲劳、静态抑制

- `static_dedup_delta_threshold`
- `static_dedup_band_similarity_threshold`
- `static_dedup_max_suppression`
- `auditory_fatigue_decay`
- `auditory_fatigue_step`

这和你之前对视觉、注意力、重复输入疲劳的哲学是一致的：不该反复把长期不变的底噪当成高价值新输入。

### 2.5 入池对象形态

当前听觉输入已经不是单一一坨，而是分成了多种 SA 形态：

- `audio_window_unit`
- `audio_global_feature_unit`
- `focus_priority_samples`
- `memory_write_samples`
- `global_structure_samples`

所以当前系统不是“完全没有听觉”，而是已经有了：

**粗频谱 + 听觉焦点 + 新异检测 + 静态去重 + 全局结构摘要**

## 3. 当前方案的主要缺口

虽然底座已经不错，但如果目标是更强的听觉识别、多模态绑定、甚至后续的语音/节奏/环境声理解，那么当前 hearing_sensor_v1 还有几个明显缺口。

### 3.1 缺少更细的“音色/谱形”通道

当前有粗 band energy 和 centroid，但还缺少：

- 频谱平坦度
- 频谱滚降
- 频谱带宽
- 频谱对比度
- 谱斜率
- 噪声感 / 音调感

这类量非常像视觉里的“边缘、形状、纹理”。  
它们不是“这就是猫叫”的硬标签，但它们会非常明显地影响相似度匹配和区分能力。

### 3.2 缺少“音高 / 和声骨架”通道

当前有 `dominant_hz`，但还不够。

还缺少：

- F0 / 基频
- voiced probability / 是否有明确发声
- pitch stability / 音高稳定性
- pitch interval / 音高跳变幅度
- chroma / 音级分布
- harmonicity / inharmonicity

否则很多语音、旋律、音色上的“像不像同一种对象”会比较难稳。

### 3.3 缺少“节奏 / 调制”层

虽然你已经给系统做了节奏感设计，但 hearing sensor 本身还缺少更原生的节奏结构提取层：

- onset envelope
- inter-onset interval
- tempogram
- modulation strength
- attack / sustain / decay 轮廓

这类信息对人声、敲击、机械音、连续背景声的区分非常重要。

### 3.4 缺少“听觉对象层”

视觉已经开始走向“结构对象 + 动态对象”的路线。  
听觉其实也需要类似的一层：

- 某个稳定的谐波团
- 某个持续的人声频带
- 某个短暂的敲击事件
- 某个持续移动的噪声团

也就是说，不应只有“一个窗口的频谱特征”，还应逐步形成：

**跨多个 tick 持续存在的 auditory object / audio pattern unit**

### 3.5 缺少“结构通道 -> 感受 -> 召回调制”的统一闭环

视觉和认知感受现在已经很接近这条闭环了。  
听觉下一步不该只是多加几个数字，而要让这些数字也能：

1. 形成听觉结构信号；
2. 超阈值后进入状态池；
3. 反向调制相关召回与注意力。

## 4. 现代音频识别路线的外部调研结论

这次我重点看的，不是“哪个黑箱模型准确率高”，而是“哪些路线适合被拆成 AP 通道”。

### 4.1 可解释特征路线：最适合做 AP 核心

这类路线以 `openSMILE`、`librosa`、经典 DSP 特征为代表。

它们的共同优点是：

1. 特征本身可解释；
2. 是连续值；
3. 能直接做向量匹配；
4. 可以自然转成 SA；
5. 对你的统一通道哲学最友好。

代表资料：

- `openSMILE` 官方枚举了标准特征集，包括 `GeMAPS/eGeMAPS/ComParE_2016` 等：[audEERING opensmile FeatureSet](https://audeering.github.io/opensmile-python/api/opensmile.FeatureSet.html)
- `GeMAPS` 论文强调使用“最小但有理论意义的参数集”，而不是纯 brute-force 大特征堆砌：[GeMAPS 论文](https://opus.bibliothek.uni-augsburg.de/opus4/files/38554/0214.pdf)

这一路线和你的哲学非常契合，因为它不是把“答案”硬编码进去，而是提供一组偏底层、连续、解释性强的观察通道。

### 4.2 预训练音频嵌入路线：适合做辅助通道，不适合直接当核心

代表：

- YAMNet
- OpenL3
- CLAP
- PANNs

这些模型能给出很强的高层表示，但问题是：

1. 太黑箱；
2. 通道意义不直接透明；
3. 很难直接映射到“哪个通道为什么增强了召回”；
4. 容易把 AP 本体变成“嵌入外壳 + AP 外围胶水”。

所以我的判断是：

**它们适合做辅助 dense manifold 通道，不适合替代 AP 的核心音频表示。**

其中：

- `YAMNet` 官方说明它基于 AudioSet，输入是 64-bin log-mel patch，并产生 1024 维 embedding 与 521 类打分：[YAMNet README](https://github.com/tensorflow/models/blob/master/research/audioset/yamnet/README.md)
- `OpenL3` 官方文档说明可直接提取逐帧音频 embedding，默认基于 AudioSet，可选 `mel128/mel256`、`512/6144` 维：[OpenL3 tutorial](https://openl3.readthedocs.io/en/stable/tutorial.html)
- `CLAP` 是音频-语言对比预训练，把文本与音频拉到同一空间里，适合做跨模态检索辅助：[CLAP arXiv](https://arxiv.org/abs/2206.04769)

### 4.3 语音 SSL 路线：很强，但现阶段不适合当 AP 听觉核心

代表：

- wav2vec 2.0
- HuBERT

这类路线在语音任务上非常强，但它们更偏：

1. 语音专用；
2. 大模型式自监督表示；
3. 黑箱程度高；
4. 工程重量大。

例如 `wav2vec 2.0` 官方摘要直接说明其目标是自监督语音表示学习与少标注 ASR 提升：[wav2vec 2.0 arXiv](https://arxiv.org/abs/2006.11477)

所以它更适合未来做：

- 语音特化辅助通道
- 语音内容强识别的外挂对照实验

而不适合在你现在这个阶段，替代 hearing sensor 的主干。

## 5. 哪些音频特征最适合拆成 AP 新版通道

下面是我认为最适合二期继续增加的通道族。

## 5.1 谱形 / 音色通道族

这类通道，是音频最接近视觉“结构纹理”的部分。

推荐优先级：**最高**

建议新增：

- spectral bandwidth
- spectral rolloff
- spectral flatness
- spectral contrast
- spectral slope / tilt
- spectral crest
- harmonic-to-noise ratio
- tonalness / noisiness
- brightness

外部依据：

- `spectral_contrast` 用峰谷能量差衡量窄带结构与宽带噪声的差异，文档明确指出高对比度更像清晰窄带信号，低对比度更像宽带噪声：[librosa spectral_contrast](https://librosa.org/doc/main/generated/librosa.feature.spectral_contrast.html)
- `spectral_flatness` 直接衡量声音更像噪声还是更像有音高的声音：[librosa spectral_flatness](https://librosa.org/doc/main/generated/librosa.feature.spectral_flatness.html)
- `spectral_rolloff` 表示频谱累计能量达到某比例时的截止频率位置：[librosa spectral_rolloff](https://librosa.org/doc/main/generated/librosa.feature.spectral_rolloff.html)

它们在 AP 里的意义：

- `flatness` 很像“粗糙/纯净感”
- `contrast` 很像“结构清晰度”
- `rolloff` 很像“明亮度上界”
- `bandwidth/slope` 很像“音色展开形状”

它们不直接说“这是苹果”，但会极大影响“这是不是同一种声音对象”的模糊召回。

## 5.2 音高 / 和声通道族

推荐优先级：**高**

建议新增：

- F0 / pitch
- voiced probability
- pitch stability
- pitch jump size
- harmonicity
- inharmonicity
- chroma / HPCP
- tonnetz

外部依据：

- `pYIN` 是经典的可解释 F0 估计方法，先给候选音高，再用 Viterbi 求最可能的 F0 序列与发声标志：[librosa.pyin](https://librosa.org/doc/main/generated/librosa.pyin.html)
- `CREPE` 代表更强的 pitch tracking 路线，论文明确指出其目标就是基频估计，并且效果可与甚至优于 pYIN：[CREPE arXiv](https://arxiv.org/abs/1802.06182)
- `chroma_cqt` 可表示 12 音级能量分布：[librosa chroma_cqt](https://librosa.org/doc/main/generated/librosa.feature.chroma_cqt.html)
- `tonnetz` 把 chroma 投影到和声关系坐标上：[librosa tonnetz](https://librosa.org/doc/main/generated/librosa.feature.tonnetz.html)

我对 AP 的建议是：

1. **第一期优先上 pYIN 风格的可解释 F0/voicing 通道**
2. chroma/tonnetz 放到“音乐/语音延伸层”，不是所有音频都强依赖它
3. CREPE 可以作为后续对照项，但不建议现在直接作为核心

## 5.3 时序 / 节奏 / 调制通道族

推荐优先级：**最高**

建议新增：

- onset envelope
- inter-onset interval
- tempogram
- modulation energy
- repetition regularity
- attack / sustain / decay descriptors

外部依据：

- `onset_strength` 直接就是事件起始强度：[librosa onset_strength](https://librosa.org/doc/main/generated/librosa.onset.onset_strength.html)
- `tempogram` 是 onset strength envelope 的局部自相关表示，适合描述局部节拍与脉冲结构：[librosa tempogram](https://librosa.org/doc/main/generated/librosa.feature.tempogram.html)

这部分和你已经建立的“节奏感哲学”尤其契合。

在 AP 里它们不仅能用于识别：

- 敲击
- 语速
- 连续机械节律
- 音乐节拍

还能自然形成：

- 节奏把握感
- 节奏期待感
- 节奏违和感

## 5.4 谐波 / 瞬态分离通道族

推荐优先级：**高**

建议新增：

- harmonic energy ratio
- percussive energy ratio
- harmonic persistence
- percussive burst density

外部依据：

- `HPSS` 能把谱图中的谐波部分与打击部分分离：[librosa HPSS](https://librosa.org/doc/main/generated/librosa.decompose.hpss.html)

这对 AP 很重要，因为它等于在听觉里引入了类似“对象层切分”的能力：

- 持续的人声/旋律
- 短促的敲击/爆破
- 背景持续嗡鸣

这些会天然形成不同的“听觉对象倾向”。

## 5.5 语音特化通道族

推荐优先级：**中**

建议新增：

- MFCC
- formant-like summary
- jitter
- shimmer
- speech rate proxy

外部依据：

- `MFCC` 仍然是音频与语音分析里最经典的压缩音色表征之一：[librosa MFCC](https://librosa.org/doc/main/generated/librosa.feature.mfcc.html)
- `GeMAPS/eGeMAPS` 本身就是偏语音/情感/副语言分析的可解释参数集：[openSMILE FeatureSet](https://audeering.github.io/opensmile-python/api/opensmile.FeatureSet.html)

我的建议是：

- 不要一开始就让“语音专用特征”支配整个听觉
- 但如果目标是人声理解、多模态语言学习，它们迟早非常有价值

## 6. 这些通道如何映射到 AP 的统一通道哲学

这是最关键的部分。

你的问题不只是“能不能提特征”，而是“这些特征是不是 AP 自己的能力”。

我的判断是：

**只要这些特征是作为连续观察通道接入 AP，并由 AP 的状态池、召回、注意力、奖惩和感受闭环来决定其意义，那么能力主体仍然是 AP。**

原因和视觉一样：

1. 通道只提供观察偏置，不提供最终标签；
2. 标签、联想、奖惩、跨模态绑定、时序回放、动作调制，都还是 AP 完成；
3. 这些通道之间会在统一状态池里混合，不是每条通道单独判决。

所以，听觉推荐采用和视觉相同的统一模式：

### 6.1 通道层

每条通道产生连续量，例如：

- `audio_timbre_flatness`
- `audio_timbre_contrast`
- `audio_pitch_f0`
- `audio_pitch_stability`
- `audio_rhythm_pulse_strength`
- `audio_harmonic_ratio`

### 6.2 感受层

当某些通道的结构显著时，形成对应“主观听觉感受”：

- 音色清晰感
- 噪声感
- 音高稳定感
- 节奏把握感
- 接近人声感
- 打击突发感

它们本质上和“惊、违和感、节奏感、时间感受”是同一哲学。

### 6.3 召回调制层

这些感受进入状态池后，反过来增强相关通道的局部匹配：

- “音高稳定感”增强对相近音高序列的召回
- “节奏把握感”增强对相近 tempogram 的召回
- “噪声感”增强对噪声型背景对象的匹配
- “人声感”增强对 formant / voiced / harmonic 相关对象的权重

这就形成了你要的统一闭环。

## 7. 针对 AP 的最终建议分层

如果按工程优先级，我建议分四层推进。

## 7.1 第一层：保留并整理当前 hearing_sensor_v1

不要推翻现有实现。

当前已有这些非常值得保留：

- RMS
- ZCR
- coarse band energy
- dominant_hz
- spectral centroid
- novelty
- onset_strength
- audio_focus
- attention_boost
- static dedup
- auditory fatigue

它们是很好的底座。

## 7.2 第二层：优先补“谱形 + 音高 + 节奏”三大主族

我建议下一轮优先新增：

1. 谱形：
   - bandwidth
   - rolloff
   - flatness
   - contrast
2. 音高：
   - F0
   - voiced probability
   - pitch stability
3. 节奏：
   - onset envelope summary
   - IOI summary
   - tempogram summary

这是性价比最高的一层。

## 7.3 第三层：建立“听觉对象层”

新增类似：

- `audio_pattern_unit`
- `audio_object_unit`
- `audio_motion_unit`

用于把连续若干 tick 的相似频谱/谐波团/瞬态团拼成一个对象。

这会极大提升：

- 背景声与主体声分离
- 人声持续追踪
- 事件边界切分
- 多 tick 累积认知

## 7.4 第四层：把预训练 embedding 当外挂辅助

如果后面要上：

- YAMNet
- OpenL3
- CLAP

建议只把它们作为：

- 辅助 dense channel
- 高层相似度参考
- 与白箱通道做对照实验

不要让它们替代 AP 的主干结构。

## 8. 我给你的最终判断

### 8.1 你的判断是对的

如果希望“正常音频识别效果更好”，确实也应该像视觉一样，加入更细的结构通道。

而且音频里是存在“结构偏好”的，只是它不表现为二维边缘和角度，而更像：

- 谱形
- 谐波堆叠
- 起音边界
- 周期性
- 调制轨迹
- 谐波/打击分离

### 8.2 最适合 AP 的，不是黑箱分类器，而是“连续可解释通道族”

最推荐的核心路线是：

1. `openSMILE / GeMAPS / eGeMAPS` 这类可解释参数思想
2. `librosa` 一类的经典音频结构特征
3. 再由 AP 自己的状态池、召回、注意力、感受来完成闭环

### 8.3 最值得优先加入的通道

我给出的优先级排序是：

1. **谱形 / 音色通道**
2. **节奏 / 调制通道**
3. **音高 / 和声通道**
4. **谐波 / 打击分离通道**
5. **语音特化通道**
6. **大模型 embedding 辅助通道**

### 8.4 对新版通用通道设计的意义

这件事的重要意义是：

它进一步证明，你的“统一通道 -> 形成感受 -> 感受入池 -> 感受反向调制召回”的哲学，不只是能用于时间感、正确感、违和感，也完全能扩展到听觉。

也就是说，后续音频新增的每个通道，都可以遵守统一规则：

1. 通道先是连续量；
2. 超阈值后形成该通道自己的主观感受；
3. 主观感受成为 SA；
4. SA 反过来增强相关通道的局部匹配。

这非常适合作为你们新版通用通道设计的继续扩展方向。

   - 纯听觉区分实验

4. 最后再把 YAMNet / OpenL3 / CLAP 作为外挂辅助通道做对照，不抢核心地位
