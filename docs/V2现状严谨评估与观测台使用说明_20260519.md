日期：2026-05-19

## 1. 一句话结论

当前 AP 二期 V2 已经不是“概念原型”，而是一个：

1. 主链闭合
2. 白箱可观测
3. 可回归验证
4. 可导入导出
5. 可继续扩展多模态与自主循环

1. 已实现完整类人语言能力
2. 已实现成熟多模态语义学习
3. 已实现图像级或三维时空级重建认知
4. 已实现成熟长期自主电脑 agent

## 4. 当前只能“初步支持”，但还不能夸大的结论

### 4.1 类人语言组织能力

现在最多只能说：

- 已出现短程语言组织雏形
- 后继优势、短期链和召回主链可以共同支撑连续输出趋势

还不能说：

- 已证明稳定长上下文组织能力
- 已证明可与现代大模型同级比较
- 已证明完整语法、语义、篇章一致性

### 4.2 多模态混合理解

现在最多只能说：

- 不同模态已进入统一 SA 主链
- 召回、状态竞争、记忆写入可以共用底座

还不能说：

- 已证明鲁棒多模态语义理解
- 已完成严格 benchmark
- 已证明大规模混合学习能力

### 4.3 教师层辅助学习

现在可以说：

- AP 主链 + teacher 审阅/奖惩/门控 的闭环已经成立

但还不能说：

- 已证明长期稳定自主学习能力
- 已证明复杂开放环境中的持续任务自进化

## 5. 当前明确还不能证明什么

以下结论现在都不该写成“已证实”：

### 5.1 完整类人语言能力

原因：

1. 当前没有长程语言 benchmark
2. 没有系统性多样本输出评测
3. 现在更接近“短程组织雏形”

### 5.2 视觉采样可以高保真重建图像

原因：

1. 当前视觉表示是 patch 级
2. 有亮度、motion、gaze、坐标等白箱属性
3. 但不是像素级生成式重建系统

### 5.3 已足够支撑完整三维时间空间认知

原因：

1. 当前有 `spacetime` 结构
2. 有 `x / y / t / has_space`
3. 但没有完整 3D 世界模型
4. 没有连续多视角重建与稳定体素/对象层

### 5.4 已完成成熟真实自主电脑 agent

原因：

1. 当前已有安全执行骨架
2. 已有反馈闭环
3. 已有持续 session
4. 但还没有成熟的长期任务管理、计划恢复、多阶段目标保持

## 7. 现在有哪些东西可以写进论文

现在比较稳、比较适合写进论文的点有：

### 7.1 统一 SA 粒度的多模态主链底座

可以写的口径：

- 文本、视觉 patch、音频 window 可以统一进入同一状态竞争与记忆主链
- 不是每种模态各走一套独立大黑箱

### 7.2 混合召回白箱化

可以写的口径：

- 一级召回不是纯 ANN 黑箱
- 而是 `posting + bigram + vector + recency + spacetime` 的混合召回
- 且每条 `Bn` 候选都可给出 `score_breakdown`

### 7.3 AP 为主体、teacher 为辅助门控与奖惩

可以写的口径：

- teacher 不替代主体认知主链
- 而是以独立审计/奖惩/门控协议挂在外层

### 7.4 持续自主 session 的白箱健康层

可以写的口径：

- `goal / context / health / recover_hint`
- 提供了不同于传统黑箱 agent 的白箱运行时语义层

### 7.5 短程语言组织雏形

可以写的口径：

- 当前原型已出现由短期记忆、后继优势、一级召回和综合预测共同支撑的短程连续语言组织现象

但这部分必须写成：

- `early evidence`
- `prototype evidence`
- `preliminary support`

而不是定论。

## 8. 前端现在可以怎么用

观测台地址：

- `http://127.0.0.1:8766`

### 8.1 最简单的使用路径

1. 打开观测台
2. 左上角文本框里输入多行文本
3. 每一行会被视为一个 tick
4. 点击：
   - `启动文本闭环`
5. 跑完后看：
   - 顶部指标卡
   - 运行列表
   - Tick 列表
   - 运行观测摘要

### 8.2 多模态测试路径

可以做：

1. 文本 + 图片文件
2. 文本 + 音频文件
3. 截图感知
4. 摄像头流
5. 麦克风流

建议先测：

1. 文本闭环
2. 截图感知
3. 再测文件型多模态

### 8.3 自主循环测试路径

在 `持续自主 Session` 区：

1. 填 `session 文本提示`
2. 设 `最大 Tick 数`
3. 选外部教师模式
4. 点：
   - `启动 session`

随后重点看：

1. 顶部 `自主会话` 状态摘要
2. `Live 结构`
3. `当前运行事件`

## 10. 跑完后应该怎么看结果

### 10.1 看语言链雏形

重点看：

1. `Tick 卡片`
2. `短期链路观察`
3. `Bn`
4. `C*`
5. `A_focus`

如果连续几 tick 的：

1. `A_focus`
2. `focus_memory`
3. `Bn`
4. `C*`

有合理连续性，就说明语言链雏形是活的。

### 10.2 看多模态是否进入同一主链

重点看：

1. `Sidecar 结构`
2. `image_packet`
3. `audio_packet`
4. `R_state`
5. `exact_memory.modalities`

如果同一 tick 里同时看到文本和视觉/音频对象进入 sidecar，就说明统一多模态底座是活的。

### 10.3 看 teacher/奖励反馈是否生效

重点看：

1. `审计`
2. `teacher_review`
3. `teacher_feedback`
4. `rules_result.action_drives`
5. `selected_actions`

### 10.4 看长期运行健康状态

重点看：

1. 顶部 `自主会话` 摘要
2. `Live 结构`
3. `运行新鲜度`
4. `当前运行事件`

## 12. 关于视觉重建、空间叠加和三维认知的当前判断

### 12.1 现在能做什么

当前视觉层已经保留：

1. patch 坐标
2. brightness
3. motion
4. gaze bonus
5. fatigue penalty
6. `spacetime.x / y / t`

因此现在可以做：

1. patch 级可视化回放
2. patch 级热区叠加
3. 多 tick 的空间热点轨迹重建

### 12.2 现在还做不到什么

当前还做不到：

1. 高保真重建原图
2. 稳定 3D 世界重建
3. 完整对象级视觉场景图

### 12.3 如果要往论文方向做，当前更好的表达方式

比起说“已经能重建图像”，更稳的说法是：

> 当前 V2 已经具备面向空间 patch 级视觉状态的白箱记录与跨 tick 热区重建能力，为后续空间场景重建与时空认知实验提供了结构化底座。

## 13. 当前最推荐的论文口径

最稳妥的总口径是：

> AP 二期 V2 已经完成了一个主链闭合、混合召回白箱化、统一多模态 SA 底座、teacher 辅助闭环、持续自主 session 健康层、可导入导出与可回归验证的研究级工程底盘。当前原型已经初步支持短程连续语言组织和统一多模态状态竞争的雏形，但尚未证明完整类人语言能力、成熟多模态语义学习或图像/三维认知重建能力。
