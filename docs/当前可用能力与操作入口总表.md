## 三、最常用命令行入口

### 1. 启动观测台

```bash
python -m observatory_v2 serve --host 127.0.0.1 --port 8766 --no-browser
```

### 2. 文本闭环

```bash
python -m observatory_v2 run-text --text "今天 天气 不错" --text "今天 天气 有点 冷" --text "算了 不说了"
```

### 3. 多模态样例

```bash
python scripts\batch_runner_v2.py --dataset config\sample_dataset_multimodal.json --label "多模态样例运行"
```

### 4. 截图感知

```bash
python -m observatory_v2 run-screen --ticks 1 --text "截图感知样例"
```

### 5. 导出 runtime

```bash
python -m observatory_v2 export-runtime --out outputs\runtime_checkpoint.json
```

### 6. 导入 runtime

```bash
python -m observatory_v2 import-runtime --in outputs\runtime_checkpoint.json
```

### 7. 从 checkpoint 继续

```bash
python -m observatory_v2 continue-from-checkpoint --in outputs\runtime_checkpoint.json --text "继续 看看" --text "再 想 一下"
```

### 8. 执行策略化 forget

```bash
python -m observatory_v2 forget --keep-latest 128
```

也支持更完整的治理参数，例如：

```bash
python -m observatory_v2 forget --keep-latest 64 --strategy score_prune --min-reality-weight 0.4 --min-total-item-energy 0.2 --protect-memory-kind teacher_feedback --max-memory-count 256 --dry-run
```

补充说明：

- 最近一次真实 forget 结果会作为 `last_forget_summary` 持久化
- 最近一次 dry-run 预演会作为 `last_forget_preview_summary` 持久化
- 两者都保存在 `outputs/live/service_runtime_state.json`
- 观测台重启、runtime checkpoint 导入后仍可恢复
- 关键 `manifest / live / checkpoint / service state` JSON 已统一切到原子替换写入

### 9. 数据集级工程化运行

```bash
python -m observatory_v2 run-dataset --dataset config/sample_dataset_pipeline.json --label "数据集管线样例"
```

## 五、当前需要特别注意的边界

### 1. 向量/ANN/时空层

当前已经接入正式骨架，但还不是最终大规模数据库部署版。

### 2. 多模态

图片、音频、截图已经接入统一主链，但仍是最小感受器版本：

1. 视觉仍是 patch 级采样
2. 听觉仍是 window 级采样
3. 已支持视频文件 lazy stream、连续截图感知、webcam 实时流、microphone 实时流；双耳定位与更高频长期设备流仍未完成

### 3. 执行器

当前已经不再只是审计壳，但默认配置仍然是安全空跑：

1. `executor_enabled = false`
2. `executor_dry_run = true`

也就是说：

- 默认不会真实操作电脑
- 但动作链路、截图链路、审计链路都已经打通
