# AP 二期环境安装与可复制运行说明

## 目标

这份说明服务于一个很实际的目标：

- 以后仓库上传到 GitHub 后，别人 `git clone` 下来可以尽可能少踩坑地装好环境
- 能先跑通核心功能
- 再按需补强视频流、FAISS、真实电脑控制

当前推荐平台：

- Windows 10/11
- Python 3.11+

## 依赖分层

### 1. 核心依赖

文件：[requirements.txt](../requirements.txt)

包含：

- `numpy`
- `pillow`
- `scipy`
- `scikit-learn`
- `psutil`
- `pyautogui`

这些足以支持：

- 文本闭环
- 图片输入
- 音频输入
- 观测台
- checkpoint / runtime / memory bundle
- 基础截图 / 基础执行器

### 2. 可选增强依赖

文件：[requirements-optional.txt](../requirements-optional.txt)

包含：

- `opencv-python`
- `sounddevice`
- `faiss-cpu`

建议用途：

- `opencv-python`：真视频文件入口与 webcam 入口
- `sounddevice`：microphone 实时流入口
- `faiss-cpu`：更正式的 ANN 向量索引能力

### 3. 开发测试依赖

文件：[requirements-dev.txt](../requirements-dev.txt)

包含：

- 核心依赖
- `pytest`

## clone 用户最常见问题

### 1. `cv2` 缺失

现象：

- 视频流入口报错
- `run-video-stream` 无法解码视频

处理：

```bash
pip install opencv-python
```

### 2. `faiss` 缺失

现象：

- 不影响核心运行
- 向量层会退化为 `numpy_flat`

处理：

```bash
pip install faiss-cpu
```

### 3. `sounddevice` 缺失或麦克风不可用

现象：

- `run-microphone-stream` 无法启动
- 麦克风入口返回 `microphone_not_available`

处理：

```bash
pip install sounddevice
```

补充说明：

- 仍需本机存在可用输入设备与驱动
- 如果只是验证文本、图片、音频文件、视频文件主链，不强制依赖麦克风设备

### 4. 截图失败

现象：

- 截图感知或自主循环无法抓屏

原因通常是：

- 当前不是图形桌面
- 远程环境禁用了抓屏
- Pillow `ImageGrab` 不可用

建议：

- 在本地图形桌面直接运行
- 先执行 [环境自检.bat](../环境自检.bat)

### 5. `pyautogui` 可导入但真实执行失败

现象：

- 真实鼠标键盘控制不可用

处理建议：

- 先用 `executor_enabled=false` / `dry_run=true`
- 观测感知和记忆闭环先跑通
- 再排查桌面权限和输入权限

## 最小验证顺序

建议 clone 用户按这个顺序验收：

1. 安装环境
2. 环境自检
3. 启动观测台
4. 运行文本样例
5. 运行截图感知样例
6. 运行视频流样例
7. 运行摄像头流或麦克风流样例（可选，依赖设备）
8. 运行自主循环样例
9. 跑测试

## 测试命令

```bash
python -m unittest discover -s tests -v
```

## 设计口径

这套方案不是追求“一次性装全所有研究依赖”，而是：

- 核心能力优先可复制
- 高级能力优雅降级
- clone 用户先跑起来
- 再逐步增强

这更符合当前 AP 二期“研究原型向专业工程底座推进”的阶段性目标。
