## 1. 这份文档的目的

这份文档用于冷保存我们已经达成一致的新视觉方案，避免后续因为上下文压缩或阶段切换而重复讨论。

本文档不是“可选创意草稿”，而是后续视觉主线实现的推荐基线。

---

## 2. 当前 V1 方案的已知局限

当前 `vision_sensor_v1` 的实现特点是：

1. 每个 tick 先把原图缩到固定重建网格。
2. 在固定网格上计算视觉 patch。
3. 从全部 patch 中选出少量 patch 进入主认知链。
4. 连续重复观察同一张图时，不会真正累积越来越多的原图级空间证据。

因此，当前 V1 更像：

- 固定分辨率重建
- 小批 patch 竞争
- 记忆重复强化

而不是：

- 原图级稀疏采样
- 注视累积
- 惊鸿一瞥与定睛一看的显著差异

---

## 3. 已确认的 V2 目标

视觉感受器 V2 的目标不是传统 OCR，也不是外挂 OCR。

目标是：

1. 纯靠 AP 的视觉采样与记忆机制，逐步形成 OCR-like 的文字识别雏形。
2. 在看到图像时，能通过连续通道特征把图像结构、尤其是文字结构，映射为可匹配、可联想的视觉记忆特征。
3. 通过与文本/动作/奖励等其它通道的时空共现，逐步学会“这类字形结构对应某个词或字符”。

---

## 4. V2 的正式设计口径

### 4.1 不走传统 OCR

视觉 V2 不直接引入成熟 OCR 作为主链路。

允许未来为了对照实验存在辅助 OCR 通道，但主实验链路的哲学应保持：

- 不先验做字符识别
- 不把图像直接翻译成文字
- 而是通过视觉结构采样、状态池、记忆、召回、共现学习来形成类 OCR 能力

### 4.2 不再先缩到固定低分辨率网格

视觉 V2 的采样基准应是原图分辨率坐标系。

也就是说：

- 原图是 `W x H`
- 每个视觉采样对象的空间位置直接落在原图坐标上
- 不先做整图固定网格缩放后再认知

### 4.3 不是“每种像素值一个 SA”

视觉对象仍然是 AP 风格的对象，但不是“颜色类别 SA”。

推荐口径：

- 每个视觉采样对象仍对应一个视觉单元对象
- 其身份主要由位置与局部采样编号决定
- 真正用于匹配的是连续属性通道，而不是离散颜色类别

建议保留的连续属性通道：

1. `x`, `y`
2. `r`, `g`, `b`
3. `brightness`
4. `local_contrast`
5. `gradient_mag`
6. `gradient_dir`
7. `edge_strength`
8. `stroke_likeness`
9. `dx_from_gaze`, `dy_from_gaze`, `dr_from_gaze`
10. 可选 `local_patch_signature`（例如 3x3 / 5x5 邻域摘要）

## 5. 每 tick 的正式预算口径

### 5.1 原始视觉采样进入统一状态池

用户已明确提出，视觉原始采样不应再压到 `32~128` 才进入状态池。

推荐口径：

- 每 tick 原图直接稀疏采样 `1024 ~ 4096`
- 这些采样对象直接进入统一状态池

这里的“统一状态池”是概念口径。  
工程上允许状态池内部存在视觉专门快路径，但不要在哲学层面把它定义成外部独立感知层。

### 5.2 A_focus / 记忆高优先区

不是所有进入状态池的视觉样本都要进入焦点记忆。

推荐口径：

- 焦点高优先区：`32 ~ 128`
- 用于：
  - A_focus 形成
  - 短期记忆 focus chain
  - exact memory 中更适合保留的视觉部分

### 5.3 预算解释

因此视觉 V2 应显式区分三个量：

1. `visual_raw_sample_budget`
   - 原图直接采样进入统一状态池的数量
   - 推荐初始值：`1024`

2. `visual_focus_priority_budget`
   - 高优先焦点样本数量
   - 推荐初始值：`64`

3. `visual_memory_write_budget`
   - 真正推荐写入更紧凑记忆签名的视觉数量
   - 推荐初始值：`32`

## 6. 注视与连续观察的正式口径

### 6.1 惊鸿一瞥 vs 定睛一看

这是 V2 的核心差异能力。

同一张图片如果只看 1~2 个 tick：

- 只有稀疏、粗糙的视觉证据
- 可形成大概轮廓和热点结构

同一张图片如果持续观察 20~50 tick：

- 视焦点附近会持续补样
- 局部结构密度明显提高
- 更容易形成可稳定联想的文字/图形结构

### 6.2 采样不是全局均匀

每 tick 的采样应带偏置，而不是全图均匀纯随机。

建议综合采样权重：

1. 距离视焦点越近，采样概率越高
2. 边缘强、局部对比度高，采样概率越高
3. 高亮区域可适度增益
4. 长期重复高频背景区域可加入疲劳抑制
5. 对已高密覆盖区可略微降低采样，以促进新证据进入

### 6.3 跨 tick 注视累积缓冲

必须引入显式的 `fixation_buffer` 或同类机制。

它的职责：

1. 记录最近若干 tick 的视觉稀疏采样点
2. 保留每个采样点的置信度、最近更新时间、与视焦点关系
3. 允许连续观察时对局部区域进行越来越密的补样
4. 支持前端“注视累积图像”重建

## 7. 关于状态池与效能的正式口径

### 7.1 理论口径

从理论上讲，状态池规模大并不必然导致每 tick 复杂度同样变大。

理想工程目标应是：

- 每 tick 真正重计算的成本主要由新增样本数决定
- 已存在对象大部分通过缓存、懒衰减、递推进行维护

### 7.2 当前工程现实

当前 `StatePoolV2` 还不是完整的增量式实现。

因此在把视觉 raw sample 提高到 `1024~4096 / tick` 之前，需要先逐步完成：

1. 懒衰减
2. 读路径缓存
3. top / anchor / focus 的增量化
4. 视觉对象的专门快路径
5. 预测包的递推缓存

### 7.3 已确认的第一阶段优化方向

已开始推进并确认合理：

1. 用懒衰减代替每 tick 全表衰减
2. 给状态池同 tick 读路径加缓存
3. 后续逐步增加：
   - 视觉快路径
   - 预测包递推缓存
   - 热点子集维护

## 8. 前端观测台必须提供的可解释性展示

为了验证这套视觉方案，前端必须能在原图上直观看到：

### 8.1 当前视焦点

要求：

- 在原图上用非常显眼的颜色点标出 gaze center
- 必须一眼可见

### 8.2 当前 tick 的原图稀疏采样点

要求：

- 在原图上叠加当 tick 采样点
- 按来源或采样原因区分颜色，例如：
  - 焦点偏置采样
  - 边缘增强采样
  - 高对比度采样
  - 普通探索采样

### 8.3 焦点高优先区 / 记忆样本

要求：

- 用更粗、更亮或不同形状标出真正进入 A_focus / 记忆高优先区的视觉样本

### 8.4 注视累积图

要求：

- 支持查看多个 tick 合并后的注视累积结果
- 让用户直接看到“越看越清楚”的空间效果

---

## 9. 关于 OCR-like 能力的实验口径

这套系统不应宣称“已经有 OCR”。

更准确的表述应是：

1. 通过原图分辨率稀疏采样与注视累积，系统有机会形成稳定字形结构记忆。
2. 通过与文本通道的时空共现，系统有机会逐步学会“某视觉结构”与“某文本单位”的对应。
3. 这是一条 AP-native 的 OCR-like 路线，而不是现成 OCR 调用。

## 10. 下一阶段推荐落地顺序

### 阶段 A：状态池底盘优化

1. 懒衰减
2. 状态池读缓存
3. top / anchor / focus 增量优化

### 阶段 B：视觉感受器 V2 基础版

1. 原图分辨率稀疏采样
2. 焦点偏置采样
3. `1024` 级 raw sample 进入统一状态池
4. `64` 级 focus priority 样本

### 阶段 C：注视累积与前端显示

1. fixation_buffer
2. 原图叠加 gaze 点
3. 原图叠加采样点
4. 注视累积图

### 阶段 D：OCR-like 验证实验

1. 图像文字与文本标签共现训练
2. 连续观察 vs 一瞥对比实验
3. 字形结构稳定召回实验

---

## 11. 一句话总结

视觉感受器 V2 的主线不是“把图缩小再选几个 patch”，而是：

**直接在原图分辨率上做连续通道的稀疏采样，让大量视觉证据进入统一状态池，再通过视焦点偏置、注视累积和记忆写入选择，逐步形成 AP 原生的 OCR-like 能力。**
