## 6. 审阅时请特别避免的误区

1. 不要拿一期最小原型直接定义当前二期仓。
2. 不要用只读到规则层的印象直接定性整个架构。
3. 不要把“没有神经网络权重矩阵”直接推出“没有任何文本组织/联想/抽象雏形”。
4. 不要把过载压力测试等同于公平能力测试。
5. 不要把“当前尚未成熟”偷换成“架构没有独特价值”。

---

## 7. 建议给外部审阅者的最低要求

如果要对 AP V2 做严肃评价，至少应当：

1. 读完纠偏说明
2. 读完 README 顶部说明
3. 查看主链代码位点
4. 至少核对 2-3 组专项实验报告与对应 summary
5. 在结论里明确区分：
   - 成立的批评
   - 证据不足的批评
   - 架构价值
   - 当前边界

否则很容易重演“拿局部模块外推整个系统”的问题。
