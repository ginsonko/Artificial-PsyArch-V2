使用说明

你现在可以怎么用

  • 最简单的入口是左上角文本框。每一行会被当成一个 tick，点“启动文本闭环”即可跑一轮最小语言链实验。
  • 想测多模态，可以直接选图片和音频，再点“启动多模态样例”；也可以点“启动截图感知”让当前屏幕进入主链。
  • 想看持续自主循环，就填写“持续自主会话”区的提示文本，点“启动自主会话”。运行中可暂停、恢复、停止，也可尝试恢复中断会话。
  • 运行后先看“实时总览”和“运行观测摘要”，再点左侧运行列表和逐拍列表往下钻。常用白箱页签是“单步卡片（逐拍）/ 旁路结构（白箱中间层）/ 运行时结构（总运行态）/ 审计 / 规则”。
  • 想改先天规则或调参器，直接在底部中文编辑器里改，先点“校验”，再点“规则试算”，确认差异后再保存。

建议测试顺序

  • 语言链雏形：输入三到五行短句，观察单步卡片里的“注意焦点（A_focus）”、短期记忆、一级召回和综合预测是否连续。
  • 多模态混合：选择图片加文本，查看旁路结构里的“图像包（image_packet）”、视觉采样块和“R_state 头部”是否同时出现。
  • 自主闭环：启动自主会话，观察实时结构（运行快照）中的阶段、健康状态、恢复提示和最近动作。
  • 规则与调参：在编辑器里改一条规则或一档调参器，使用“规则试算”对照真实执行和草稿差异。
  • 排查问题：先看右侧“运行与报错日志”，再看实时结构、运行事件、规则提醒和调参提醒。

核心术语

  • `A_focus`：当前 tick 真正赢得注意力的少数对象，也就是这一拍最显著、最可能形成记忆的内容。
  • `Bn`：一级记忆召回候选，表示当前状态最先拉回来的记忆壳或记忆条目。
  • `C_i`：从一级召回继续展开的预测分支，主要承载后继优势、邻近时空扩展和虚能量传播。
  • `C*`：当前综合后的虚能量预测包，是这一拍“系统觉得接下来可能会发生什么”的白箱展开。
  • `Sidecar`：旁路结构，也就是“白箱中间层”，保存当前 tick 的中间结果，便于审计，但不等于状态池本体。
  • `Runtime`：运行时总状态，也就是“总运行态”，汇总状态池、短期记忆、向量索引、执行器和学习器的整体快照。
